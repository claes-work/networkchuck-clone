# installing my "FREE" cloud phone system (AWS and 3CX)

## Summary

Chuck builds a fully functional business phone system entirely in the cloud, running
a [[wiki/entities/3cx|3CX]] PBX on the AWS free tier and giving it a real, dialable
phone number via Amazon Chime's SIP trunk service. He frames it as "mostly free": AWS
and 3CX are free for 12 months, but the phone number itself costs money like a cell
phone. The video is an end-to-end tutorial that walks through three components — the
AWS free-tier host, the 3CX phone system, and an Amazon Chime SIP trunk — and shows
soft phones (the 3CX app on his cell and his daughter's iPad), inbound and outbound
calling, phone-number normalization, and an IVR ("digital receptionist"). It builds
directly on an earlier video where he ran the same 3CX PBX on a Raspberry Pi 4, and it
leans on his voice/collaboration background. The tone is playful and heavy on his
recurring "hack the YouTube algorithm (ethically)" and "coffee break" bits.

## Key claims (dated — the build)

_All paraphrase unless quoted; from the 2021-04-22 build._

- The project has three components: the AWS free tier (cloud host), 3CX (the phone
  system, and the video's sponsor), and a real phone number via Amazon Chime's SIP
  trunk service.
- It is "mostly free": AWS free tier and 3CX are free for 12 months; Amazon Chime
  requires paying for a phone number, similar to a cell phone plan.
- **AWS setup:** create a free-tier AWS account (free for 12 months, some things always
  free, credit card required but they stay within free-tier limits), then do three
  things — create an IAM user, create an EC2 key pair, and subscribe to a Debian image.
- The IAM user (named `3cx`, programmatic access) is given the `AmazonEC2FullAccess`
  policy so that 3CX itself can log into AWS and deploy the server; its credentials are
  downloaded as a CSV.
- A key pair (named `3cx key pair`, PEM format) is created in EC2 — it functions as a
  more secure password for logging into the virtual machine, and the downloaded file
  cannot be recovered if lost.
- Region matters: Chuck uses `us-east-1` (North Virginia) and stresses picking the
  region closest to you and remembering it, because 3CX needs the matching region later.
- Via AWS Marketplace he subscribes to **Debian 9**, the free Linux distro 3CX uses to
  spin up the server; the EC2 instance stays free as long as it uses the `t2.micro`
  size (later the 3CX wizard defaults the host to `t2.nano`).
- **3CX setup:** sign up for a free 3CX account (free for one year), choose self-hosting
  in your own cloud, set locale/time zone, name the system and domain, and pick an
  extension length (Chuck likes four digits).
- For the hosting provider Chuck selects Amazon AWS and pastes in the IAM user's access
  key ID and secret from the CSV; 3CX then accesses the AWS console and deploys the
  server itself (using the selected region and the `3cx key pair`).
- Deployment credentials shown by 3CX must be saved because they are hard to recover;
  the server (his `networkchuck.my3cx.us` PBX) provisions in the cloud in a few minutes.
- Soft phones are added by scanning a QR code in the 3CX app — Chuck registers his cell
  phone and adds extension `1001` for his daughter Chloe on her iPad, then places a
  working video call between them through the cloud PBX.
- A real phone number requires a **SIP trunk** (Session Initiation Protocol) — the
  connection that lets outside callers reach the PBX and lets the PBX present the number
  as caller ID. This is where the free part stops.
- Chuck uses **Amazon Chime** as the SIP trunk provider to keep server and trunk cost on
  one AWS account; he buys a toll-free (888) number through Chime's phone-number
  management.
- A Chime **voice connector** ("network trunk", region us-east, encryption disabled per
  3CX's requirement) links Chime to the 3CX PBX.
- **Outbound (termination):** enable termination, add the PBX's public IPv4 address (as
  a `/32`) to the allowed-host list, set the calling plan (United States), and create
  credentials that 3CX uses to authenticate to the trunk.
- **Inbound (origination):** enable origination, add an inbound route pointing to the
  3CX FQDN on port `5060` over TCP (priority/weight left at the example values), then
  assign the purchased number to the trunk from inventory.
- On the 3CX side he adds a SIP trunk (country US, provider "Amazon Chime Voice
  Connector"), sets the main trunk number in **E.164** format (`+` and country code `1`
  for the US), and fills in the Chime outbound host name as the registrar plus the Chime
  username/password.
- Inbound calling is tested by calling the toll-free number and having it ring Chloe's
  iPad — it works.
- **Outbound rule:** calls dialed with a leading `9` route out through the Amazon Chime
  voice connector. Chuck demonstrates **phone number normalization** — strip the leading
  `9`, then prepend `+1` — so Amazon receives the number in E.164 format; outbound
  caller ID is set to the purchased number. He test-calls IT Pro TV to prove it works.
- **IVR / digital receptionist:** he adds a "main line" on extension `8000`, records the
  greeting prompt by having 3CX call his extension, and configures menu options (some
  just play a prompt and exit; option four connects to an extension for his new employee
  Austin). An inbound rule routes the toll-free DID to the IVR.
- Teaser of future capabilities not covered: 3CX messaging, Facebook/SMS/live-chat
  integration, website live chat, and call queuing.
- You can do this whole build for free (except the toll-free number); a local DID can be
  used instead of toll-free, and it works from anywhere — no Raspberry Pi or any hardware
  required, unlike his earlier Pi build. Desk/hard phones (e.g. the Cisco phones from his
  Raspberry Pi video, and other brands on 3CX's compatibility list) can also be added.
