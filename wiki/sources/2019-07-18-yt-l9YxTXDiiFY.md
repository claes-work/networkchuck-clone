# EVERYONE needs to learn LINUX - ft. Raspberry Pi 4

## Summary

An early (2019) NetworkChuck video arguing that everyone in IT — help desk, network
engineer, system engineer, programmer, aspiring hacker — must learn Linux, and that
the then-newly-released Raspberry Pi 4 (~$35) is his favorite cheap, hands-on way to
start. Chuck frames Linux as an ordinary operating system (the manager of a device's
hardware resources) that already runs the world: smartphones, Teslas, smart-home
devices, and the vast majority of cloud servers and top websites. His core pitch is
that Linux is free and open source, that it "runs on everything," and that precisely
*because* it looks scary (a black terminal screen, no GUI) most IT engineers avoid it —
which makes knowing it a rare, high-value, resume-boosting skill.

The bulk of the video is a beginner walkthrough: he shows three free ways to get a
Linux environment (a Raspberry Pi with Raspbian, a free Microsoft Azure Ubuntu server,
and WSL / Ubuntu on Windows 10 via the Microsoft Store), then walks the viewer through
installing and starting an Apache web server and editing the default `index.html` — the
whole thing done alongside his young daughter (self-reported; name withheld per repo
policy of keeping the subject's children unnamed) to prove it isn't scary.
He closes by mapping the path forward: earn the CompTIA Linux+ (or LPIC) certification,
then move toward DevOps and network automation. He publicly commits to earning Linux+
himself and asks viewers to hold him accountable. Recurring guest reference: Linux
expert/instructor Shawn Powers, whose CBT Nuggets course and career story Chuck cites.

## Key claims (dated — his argument)

All dated 2019-07-18, paraphrased unless quoted:

- Everyone in IT needs to learn Linux — it is now a required, expected skill regardless
  of role (help desk, network/system engineer, programmer, hacker).
- Learn it now and you'll be rare; wait too long and you'll get left behind. The earlier
  the better — he says he's already teaching his own kids.
- Linux is just an operating system: the manager of a device's hardware resources,
  sitting between apps and hardware — not inherently special or intimidating.
- Linux is the most popular OS in the world by far and runs almost everything:
  smartphones, laptops, cars (Teslas), smart-home devices ("even your toilet"), Android,
  and most cloud servers — even Microsoft's Azure runs Linux.
- He claims 96% of the top 1 million websites run on Linux, and that YouTube and Netflix
  run on Linux servers.
- The reason everyone uses Linux is that it's free and open source — you can read,
  change, and edit the code, which is why there are so many "flavors" (distros).
- Two reasons an IT person should learn it: (1) it runs on everything; (2) it's scary,
  and that scariness is exactly why it's valuable — the barrier keeps most engineers
  away, making it a rare skill you can demand money for.
- Putting Linux on your resume gives instant "street cred" and makes you employable for
  adjacent jobs. He cites Shawn Powers landing a Windows DBA job with no Windows DBA
  experience because he knew Linux, and his own hire as a college database manager
  (Microsoft SQL) on the strength of being an all-around IT guy with Linux experience.
- The learning curve is a small "bump in the road," not a mountain — get past the initial
  discomfort of the command line and it's smooth sailing.
- The best way to learn is hands-on, as soon as possible. His favorite method: a
  Raspberry Pi 4 (about the size of a credit card, ~$35), which just came out — throw
  Linux on it and start learning (also runs Minecraft, Python, etc.).
- Three free/cheap starting environments: Raspberry Pi (his favorite), a free Azure
  Ubuntu server (also teaches you cloud), or WSL/Ubuntu installed on Windows 10 via
  PowerShell + the Microsoft Store (via the Microsoft–Canonical partnership).
- A practical first project builds confidence: `sudo apt-get update`, install Apache
  (`sudo apt install apache2`), start the service, hit `localhost`, then edit the default
  `index.html` (cd to `/var/www/html`, `rm`, `nano`) — after which "you know Linux now."
- Next steps to a career: earn CompTIA Linux+ (recently renewed) or LPIC to become a
  Linux admin, then move into DevOps — every DevOps tool relies on Linux in some regard.
- Specifically for network engineers: network automation and network programmability
  require Linux; getting into the cloud (Azure, AWS) requires Linux.
- He publicly commits to earning Linux+ himself and asks the audience to hold him
  accountable and learn alongside him. Signals he'll be making more Linux content.
