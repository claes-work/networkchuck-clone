# Python Logical Operators (the fun stuff!!)

## Summary

Episode six of NetworkChuck's "Python right now" free course series. The lesson
teaches Python logical operators (`or`, `and`, `not`) and the boolean data type,
carried on the running "coffee shop bouncer" narrative: a Python script that
decides whether a person may enter based on their name and whether they are evil.
Prior episodes filtered a single villain ("Ben"); this episode extends the script
to also block "Patricia" and "Loki" using logical operators.

Chuck introduces `or` first (block the person if the name equals Ben **or**
Patricia **or** Loki), chains multiple `or`s, then fixes a hardcoded print message
by concatenating the `name` variable so the rejection greets the actual person.
He then introduces `and` via a more complex requirement: an evil person may still
enter if they have done four or more good deeds that day — expressed as
`if evil_status == "yes" and good_deeds < 4:`. He uses a flowchart to make the
branching logic visible, and notes that `input()` returns a string by default so
the good-deeds count must be wrapped in `int()` for the numeric comparison.

Finally he covers the `not` operator on a stripped-down "clean slate" script
(`if not name == "Ben":`), acknowledging it is brain-bending and something he
uses less often. He demonstrates the boolean data type interactively in the
Python terminal (setting a variable, evaluating `==` to get `True`/`False`,
checking `type()` returns `bool`), shows `not me == "Nick"` flipping `False` to
`True`, and closes by recommending the more common, more readable `!=` comparison
operator in place of `not ==`. Every episode ships with a free in-browser Python
lab; the series is sponsored by ITProTV.

## Key claims (dated — concepts)

- **[2022-03-12]** Logical operators let a single `if` statement evaluate more
  than one condition; the three covered are `or`, `and`, and `not`. (paraphrase)
- **[2022-03-12]** `or` evaluates true when at least one of its conditions is
  true — e.g. block the visitor if `name == "Ben" or name == "Patricia"`; you can
  chain additional `or` clauses (adding Loki as a third). (paraphrase)
- **[2022-03-12]** `and` requires **both** (all) of its conditions to be true
  before the branch runs — used so a person is rejected only when they are evil
  **and** have done fewer than four good deeds. (paraphrase)
- **[2022-03-12]** `not` inverts a condition: `if not name == "Ben"` runs the
  branch when the name is anything other than Ben — the opposite of the original
  statement. (paraphrase)
- **[2022-03-12]** `True` and `False` are booleans, a distinct Python data type
  (`bool`), joining the earlier-covered strings, integers, and floats; `type()`
  on a comparison expression returns `bool`. (paraphrase)
- **[2022-03-12]** A comparison with `==` evaluates to a boolean (`True`/`False`);
  Python can be run interactively line-by-line in the terminal to test this.
  (paraphrase)
- **[2022-03-12]** `input()` returns a string by default, so a numeric answer
  (good-deeds count) must be wrapped in `int()` before a numeric comparison like
  `< 4`. (paraphrase)
- **[2022-03-12]** The comparison operator `!=` ("not equal") accomplishes the
  same result as `not ==` and is the more common, more readable way to write it —
  Chuck's preferred form. (paraphrase)
- **[2022-03-12]** A flowchart helps reason through nested conditional logic
  before writing the code. (paraphrase)
- **[2022-03-12]** Repetition cements programming concepts; extra challenge labs
  are provided to reinforce the lesson. (paraphrase)
