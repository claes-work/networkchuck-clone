# what is an IP Address? // You SUCK at Subnetting // EP 1

## Summary

The kickoff episode of Chuck Keith's "You SUCK at Subnetting" series — a dedicated
subnetting mini-course pitched as a core, acquired skill for every IT job (networking,
ethical hacking, security, cloud), and part of his CCNA series. The episode teaches the
fundamentals of IP addressing using heavy analogies. Chuck frames an IP address as the
"phone number" of a device: every device needs one to talk to other devices or reach the
internet. He shows how to find your own IP (`ipconfig` on Windows, `ifconfig` on
Linux, wifi settings on a phone) and introduces the IPv4 address alongside its two
companions on the same screen: the subnet mask ("Mr. Subnet mask" / net mask) and the
default gateway ("Miss default gateway" / default router / router).

He explains that the router — nicknamed "Oprah" — hands out IP addresses to every device
via DHCP, which is why most home networks start with `192.168.1`. The subnet mask is
introduced with a beginner "hack": wherever an octet is `255`, that corresponding octet
in the IP address is locked and never changes across the network (the network portion);
wherever the mask is `0`, that octet can be anything 0–255 (the host portion). Analogy:
network portion = the street ("Private Drive"), host portion = the house number — devices
on the same street/network can talk directly, while reaching a device on another network
(e.g., Netflix) requires handing traffic to the default gateway/router (the "UPS"). The
episode closes with a challenge problem computing usable addresses on a `/24`
(`255.255.255.0`) network: 256 total minus 3 reserved (network address, broadcast address,
and the router's own address) = 253 usable. Includes sponsor reads for Boson (CCNA/CCNP
courseware, labs, practice exams) and Manscaped.

## Key claims (paraphrase unless quoted)

- 2022-04-05 — This is EP 1 of the "You SUCK at Subnetting" series; the series will build
  from "what is an IP address" up to full subnetting mastery, and is part of Chuck's CCNA
  series.
- 2022-04-05 — IP/subnetting knowledge is an essential acquired skill for effectively
  every IT job: networking, ethical hacking, security, and cloud all involve IP addresses.
- 2022-04-05 — An IP (Internet Protocol) address is like a phone number assigned to a
  device; without one, a device cannot communicate with other devices or connect to the
  internet.
- 2022-04-05 — You can find your device's IP address with `ipconfig` (Windows CMD),
  `ifconfig` (Linux terminal), or via wifi settings on a phone; on Windows it is labeled
  "IPv4 Address."
- 2022-04-05 — An IPv4 address is four numbers separated by three dots; each section is
  called an octet, and each octet can be any number between 0 and 255.
- 2022-04-05 — On the same screen as the IP address sit the subnet mask (aka net mask)
  and the default gateway (aka default router / router).
- 2022-04-05 — Routers assign IP addresses to devices as they join the network via DHCP;
  this is why most home networks default to starting with `192.168.1`.
- 2022-04-05 — Beginner hack: where a subnet mask octet is `255`, the corresponding IP
  octet is locked and stays the same for every device in the network; where the mask octet
  is `0`, that IP octet can be any value 0–255.
- 2022-04-05 — The locked octets are the "network portion" of the IP address; the changing
  octet(s) are the "host portion." Devices on a network are called hosts.
- 2022-04-05 — Analogy: the network portion is the street you live on (e.g., "Private
  Drive"); the host portion is your house number. Devices on the same network can reach
  each other directly; reaching a device on a different network requires the default
  gateway.
- 2022-04-05 — When a device wants to talk to something outside its own network, it must
  send the traffic to its router / default gateway to get out.
- 2022-04-05 — On a `192.168.1.0` network with a `255.255.255.0` subnet mask, there are
  256 total addresses (0–255 in the last octet), but two are always reserved: the first
  address (`192.168.1.0`, the network address) and the last address (`192.168.1.255`, the
  broadcast address, which broadcasts anything sent to it to all devices).
- 2022-04-05 — A router also consumes one address, so the example network yields 256 − 3 =
  253 usable IP addresses.
- 2022-04-05 — Chuck estimates (his own opinion, not a cited statistic) that roughly 95%
  of all networks look like this simple `192.168.1.x` example.
