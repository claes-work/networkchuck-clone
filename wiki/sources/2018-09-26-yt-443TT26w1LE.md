# Network Engineers and AWS (Amazon Web Services) FEAT. Anthony Sequeira | CCNA | CCENT

## Summary

An early NetworkChuck interview, recorded at Cisco Live, in which Chuck sits down with
fellow CBT Nuggets trainer **Anthony Sequeira** (a CCIE networking veteran who has moved
into AWS training) to make the case that network engineers should start learning cloud —
specifically AWS. Chuck frames cloud literacy as increasingly non-optional for Cisco-track
engineers ("you're gonna be in trouble when your favorite Cisco platforms move there"),
then hands most of the substance to Sequeira, who explains what AWS is (public cloud,
market leader), why companies adopt it (cost-effective access to compute/ML horsepower,
"technology democratization"), and lays out a concrete certification path for a networking
person: CompTIA Cloud Essentials / Cloud+ (vendor-neutral) → AWS Certified Cloud
Practitioner → Solutions Architect Associate → (optionally) Advanced Networking specialty.
Sequeira also fields Chuck's audience-specific questions about where AWS fits relative to
CCENT/ICND1, CCNA, and CCIE study. The video is notable as an early instance of Chuck's own
"network engineers need cloud" framing, which becomes a recurring theme in his later cloud
advocacy — though at this stage he positions himself as the collaboration/Cisco guy and
Sequeira as the AWS expert.

## Key claims (dated)

All dated 2018-09-26 (source publication date). Attribution labeled per speaker.

- **[Chuck framing]** Cloud literacy is "incredibly important" these days; network engineers
  will be "in trouble when your favorite Cisco platforms move there," and should at least
  know the basics of where those platforms are located. (paraphrase)
- **[Chuck framing]** AWS is public cloud and the market leader; "everything we know and
  love — servers, storage, networking" — is being moved into the cloud, so network engineers
  "better start learning some AWS." (paraphrase)
- **[Chuck framing]** More and more companies are moving infrastructure to the cloud, often
  starting with hybrid deployments; Chuck reports his own peers saying they're having to
  learn AWS. (paraphrase)
- **[Chuck framing]** Good news for network engineers: AWS wants them and has a whole
  certification track "just like Cisco." (paraphrase)
- **[Chuck framing]** Recurring theme across his interviews: passion is the key — you have
  to fall in love with the discipline or you'll struggle. (paraphrase)
- **[Sequeira-guest]** AWS is public cloud; exciting because it gives companies
  cost-effective access to compute horsepower (e.g., machine learning) they lack in-house —
  "technology democratization." (paraphrase)
- **[Sequeira-guest]** CBT Nuggets itself uses AWS heavily, having started with the classic
  hybrid model and moved increasingly to public cloud; many companies follow this "baby
  steps" adoption pattern. (paraphrase)
- **[Sequeira-guest]** AWS previously had no cert matching its free "AWS Technical
  Essentials" course; it now offers **AWS Certified Cloud Practitioner** as the foundational
  cert underpinning specialty tracks (architecture, operations, big data, security, advanced
  networking). (paraphrase)
- **[Sequeira-guest]** Recommended path for a network engineer: (optionally) CompTIA Cloud
  Essentials / Cloud+ for vendor-neutral fundamentals → AWS Cloud Practitioner → Solutions
  Architect Associate → consider the Advanced Networking specialty (jumping straight to
  Advanced Networking is "a stretch"). (paraphrase)
- **[Sequeira-guest]** Start with AWS because it is the market leader; skills transfer easily
  to #2 Azure (mostly a matter of renamed trademarked terms, e.g. "virtual private cloud").
  He does not believe Azure will ever catch AWS but expects it to keep gaining share. (paraphrase)
- **[Sequeira-guest]** AWS Solutions Architect salaries are near the top of salary surveys;
  you can "crush it" financially in any Cisco or AWS track if you're passionate about it. (paraphrase)
- **[Sequeira-guest]** ICND1/CCENT-level knowledge of core network components is enough to
  make the leap to cloud. (paraphrase)
- **[Sequeira-guest]** Advice to CCIE aspirants: don't get distracted by "sexy" AWS tech —
  entrench in Cisco, finish the CCIE, then dabble in public cloud (as he did himself). (paraphrase)
