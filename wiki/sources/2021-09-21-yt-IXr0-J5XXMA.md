## Key claims (dated — concepts)

All dated 2021-09-21 (paraphrase unless quoted):

- A **comment** is created with `#` (hashtag / pound sign) in front of a line; the computer
  ignores it, so comments are for humans, not the machine. Commenting out code is a handy way
  to disable lines while testing.
- `print()` is a function; whatever string goes between its parentheses (in quotes) is what
  the computer outputs.
- `input()` is a function that takes input from the user, and **automatically converts that
  input to a string** (a detail Chuck flags as important for later).
- Putting a string inside `input()`'s parentheses makes it display that string as a prompt,
  then pause and wait for the user to type a response.
- You can nest functions — e.g. `print(input())` puts the input function inside the print
  function — though printing raw input straight back is useless for the barista use case.
- A **variable** is a method of storing data for reuse. You create one with `name = value`
  (no parentheses, no quotes around the variable name); a string value goes in quotes.
- Creating a variable alone produces no output — the computer is literal and only does exactly
  what you tell it; you must `print()` the variable to see it.
- A variable's value can **vary** — reassigning it (e.g. `name = "iron man"` after
  `name = "network chuck"`) changes what prints.
- You can store user input in a variable directly: `name = input("What is your name?")`.
- Strings **and variables** can be added together with `+` (concatenation, extended from EP1
  where only strings were concatenated); you must include literal spaces inside strings
  because the computer prints exactly what you give it.
- `\n` is the **line break / new line character**, useful for cleaning up prompt formatting
  (e.g. moving the user's typed answer to a new line after a question).
- The episode's two headline takeaways: the `input()` function (requesting data from the
  user) and variables (remembering that input to use later).
