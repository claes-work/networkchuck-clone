# "Codex Lives In PowerShell Now"

## Summary

Chuck argues that while most AI tools can write PowerShell, few actually run natively inside it. He demos Codex on Windows executing a task ("how many files are in this directory?") by running real PowerShell commands by default, rather than emulating a Linux environment. His framing: for Windows sysadmins managing Active Directory, domain controllers, and Azure, Codex "lives in" the Windows environment as a local rather than "pretending to be Windows in a Linux costume," making it more integrated for real jobs.

## Key points (paraphrase)

- 2026-06-02: Distinction drawn — most AI tools can write PowerShell, but not all of them live in / run inside PowerShell.
- 2026-06-02: Relevant for Windows daily-driver users and Windows sysadmins managing Active Directory, Windows domain controllers, and Azure.
- 2026-06-02: Demo: asking Codex to count files in a directory triggers it to run PowerShell commands by default, the way a human admin would.
- 2026-06-02: Framing — Codex is "a local" in Windows, not visiting or emulating it in "a Linux costume," so it integrates more deeply into sysadmin work.
