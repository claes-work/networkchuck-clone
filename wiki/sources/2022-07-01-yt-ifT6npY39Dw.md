# ditch Google!! (build your own PRIVATE search engine)

## Summary

Chuck argues that Google and Microsoft build advertising profiles from every search
you make and sell that data, leaving you with no privacy — and that neither incognito
mode nor DuckDuckGo (which he says has a "weird relationship with Microsoft") is a real
fix. His answer is to self-host your own meta-search engine using **SearXNG** (a fork
of Searx), a privacy-focused, open-source engine that queries Google, Bing, DuckDuckGo
and 70+ other engines on your behalf. The tutorial deploys it with **Docker** and
**Docker Compose** on a Linode cloud server (Ubuntu 20.04), but Chuck stresses it runs
on anything — a spare PC, a Raspberry Pi, a NAS, or the machine you're using right now.
He walks through spinning up the server, installing Docker, cloning the SearXNG-Docker
repo, editing the `.env` file for a domain and Let's Encrypt SSL certificate, generating
a secret key, and bringing up three containers (Caddy, SearXNG, Redis) with a single
`docker-compose up -d`. He then shows settings tweaks (switching the HTTP method to
`get`, naming the instance, enabling strict safe search and autocomplete). Throughout,
the recurring theme is own-your-data privacy: the only way to get "ultimate privacy" is
to run it yourself and trust no company. He even warns that his own public instance
(search.network chuck.coffee) is more private than Google but still under his control,
so he *could* look at logs — hence build your own.

## Key claims

- (2022-07-01) Every time you search, Google and Microsoft build a profile of
  information about you and sell it to advertisers, leaving you with no privacy.
- (2022-07-01) Incognito mode does not protect your search privacy.
- (2022-07-01) DuckDuckGo is not a real privacy solution because it has a "weird
  relationship with Microsoft" (paraphrase of Chuck's claim).
- (2022-07-01) The best way to ensure privacy/anonymity online is to build your own
  search engine — and a public instance you don't control (even Chuck's own) is only
  "pretty private," not ultimate privacy.
- (2022-07-01) The tool deployed is **SearXNG**, a fork of the original **Searx**, an
  open-source **meta-search engine** — it queries Google, Bing, DuckDuckGo and other
  engines all at once and forwards the aggregated results.
- (2022-07-01) SearXNG protects privacy in three ways: (1) it builds a random search
  profile for every single search, unrelated to you, so upstream engines never tie a
  query to your identity; (2) it serves no ads; (3) it does not share your profile/data
  with the resulting website you click through to.
- (2022-07-01) Added benefit of hosting on a cloud server: searches go out under the
  server's public IP, not your home IP — so Google/Bing don't learn where you live;
  your home IP, ISP, and DNS provider otherwise reveal things about you.
- (2022-07-01) Requirements: any machine that can run Docker (Windows, Mac, or Linux),
  optionally a domain name, and (Chuck's running joke) coffee.
- (2022-07-01) Setup steps: create a Linode server (Ubuntu 20.04, ~$5/month) → SSH in →
  `apt update` / `apt upgrade` (best practice, optional) → `apt install docker.io -y` →
  `apt install docker-compose -y` → `git clone` the SearXNG-Docker repo → `cd` in →
  edit the hidden `.env` file (domain + email for Let's Encrypt SSL) → run the provided
  command to generate a secret key → `docker-compose up -d`.
- (2022-07-01) The deployment builds three Docker containers — Caddy, SearXNG, and
  Redis — visible via `docker ps`; Caddy handles the Let's Encrypt SSL certificate.
- (2022-07-01) Bringing the stack down and back up (`docker-compose down` then `up`)
  pulls the latest container versions, keeping the instance up to date.
- (2022-07-01) You can run it as a public instance (share with friends/family) or a
  private instance only reachable on your LAN; a home deployment on a local VM (Chuck
  demos Kali Linux) uses `localhost` by default, but then searches use your own IP so
  that part is not private.
- (2022-07-01) `settings.yml` in the `searxng` folder controls almost everything: change
  the HTTP `method` from the default `post` to `get` (fixes jarring back-button
  behavior), set the instance name, enable metrics/stats, enable strict safe search
  (option 2 — good for kids), and set an autocomplete engine (Chuck uses DuckDuckGo).
- (2022-07-01) SearXNG supports 70+ search engines you can enable/disable, can search
  added SQL databases (e.g. a local MySQL instance), and supports per-search proxies
  including Tor for extra anonymity; it also exposes stats and an API.
