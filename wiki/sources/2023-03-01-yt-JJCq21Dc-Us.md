# You NEED to learn Machine Learning, RIGHT NOW!!

## Summary

This is NetworkChuck's **first major AI/ML video** — the moment the channel applies its
signature "you NEED to learn X, RIGHT NOW" format to Machine Learning, opening what
becomes the channel's ongoing AI-tools content era. Published March 2023, at the height
of the early ChatGPT hype wave, Chuck pitches ML as a learnable, future-proof, and
lucrative career path that requires **no degree and no math-genius background**, and
frames the whole video around a hands-on project: building an ML model that predicts
how badly you'll lose a game of League of Legends by pulling match data from the Riot
Games API.

Chuck is explicit that he is **not** a machine learning engineer himself, so he brings
in two experts as on-camera guides: **Santiago** (an ML engineer working on computer
vision for Boston Dynamics' Spot robot) and **Nacho** (an ML engineer who built the
League of Legends workshop labs). The tutorial project runs entirely on Oracle Cloud
Infrastructure (OCI) — the sponsor — using free-tier resources and a $300 credit, with
infrastructure deployed via Terraform and Ansible, and model work done in Jupyter
notebooks with real data-science tooling (pandas, NumPy, matplotlib, scikit-learn,
AutoGluon).

The video's teaching spine is an **8-step learning path** to becoming an ML engineer,
interleaved with the OCI League of Legends labs. It is both a conceptual explainer (what
ML actually is, how a model is trained) and a "just jump in and stink at it" call to
action — Chuck's recurring learning philosophy applied to a brand-new domain for the
channel.

## Key claims (paraphrase, dated 2023-03-01)

- **Machine learning is teaching a computer to learn from data**, rather than a developer
  hand-crafting explicit rules. (Santiago) In traditional programming you write rules
  (loops, conditions) to, e.g., play tic-tac-toe; ML instead takes many samples (e.g. a
  year of temperature data) and lets the computer derive the rules automatically to
  predict future outcomes.
- **A machine learning model is just a file** that has been trained to recognize certain
  patterns.
- **Training workflow** (illustrated with a "is this a photo of NetworkChuck?"
  classifier): (1) define what you want to teach; (2) gather lots of labeled data —
  including counter-examples of "not me" so the model learns what to ignore; (3) choose a
  learning algorithm; (4) feed the data so the model weights features (beard, "crazy
  eyes," man bun) up or down; (5) test with held-out testing data to measure accuracy.
- A **convolutional neural network** is a suitable algorithm for the image-recognition
  example. Chuck admits he does not know how it works internally, only that it fits the
  problem.
- If a model's accuracy is poor (e.g. ~10%), improving it is the core skill of the ML
  engineer — learning what to change to make the model more accurate.
- **You do NOT need a degree, and you do NOT need to be a math genius** — ML can be
  learned for free.
- **Best way to start is to just start doing it** — jump in the deep end, then unpack the
  pieces you don't understand (e.g. go watch a video on the chain rule when you hit it).
  (Santiago's recommended learning style, endorsed by Chuck.)
- **Data extraction / preparation is ~70–80% of the ML engineer's job** — getting the
  right data and shaping it cleanly so the algorithm can learn well. (per Santiago and
  Nacho)
- **You need an intermediate level of Python** — not full software-developer skill, but
  scripting ability. Python is "the darling" language of data science and ML. Chuck notes
  his own free Python course on YouTube as a resource, alongside paid options like ITPro.TV
  and Codecademy.
- **Only high-school-level math is required** to get into ML — statistics, probability,
  and calculus to brush up on; you can pick up more as you need it. (Santiago) Nacho
  agrees you do not need to go back to college or hold a PhD to use ML tools. Many ML
  tools abstract the math away, but understanding it helps you know why you use each
  algorithm.
- **Recommended structured learning resources**: Brilliant's data-science path (for
  absolute beginners), Kaggle's free "Intro to Machine Learning" course, and Andrew Ng's
  Machine Learning Specialization on Coursera (three courses; the previous version was the
  most-enrolled ML course ever, redone the prior year).
- **Real tools matter**: the labs use tools that real data scientists and ML engineers
  actually use — Jupyter notebooks, scikit-learn, AutoGluon, and even building a neural
  network yourself.
- **Practice by repetition and by extending the example**: add features, get more/better
  data, try different algorithms; apply it to another game with an API; and use Kaggle
  competitions (which supply the data sets and offer prize money) to keep sharpening
  skills, then build a portfolio and look for junior roles.
- The League of Legends predictor works by **identifying match features** (champion
  chosen, lane, gold earned, minions killed, etc.) and feeding historical pro-match data,
  pulled from the Riot Games API, into the model to predict the winner.
