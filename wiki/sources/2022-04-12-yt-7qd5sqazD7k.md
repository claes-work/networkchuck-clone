# BASH scripting will change your life

## Summary

Episode two of NetworkChuck's Bash scripting series (companion to his full Bash
course on NetworkChuck Academy). Chuck frames Bash scripting as both a career
superpower (it "will make you an awesome IT engineer") and a daily
productivity/life upgrade, then teaches core scripting concepts hands-on by
building a motivational "Bash Butler" script called `best day ever.sh`.

The lesson is structured as a series of build-it-yourself challenges punctuated
by "coffee breaks." Starting from episode-one knowledge (shebang, `echo`,
`sleep`, `chmod +x`), he introduces: **variables** (set once, reused everywhere
via `$name`), three ways to populate a variable — manual assignment, `read`
(interactive user input), and **positional parameters/arguments** (`$1`, `$2`
passed on the command line) — and finally **command substitution**
(`variable=$(command)`) to store the output of commands like `whoami`, `pwd`,
and `date` inside variables for use in the script. The finished script (not
fully shown; lives on NetworkChuck Academy) also reports private IP, public IP,
weather for a chosen city, and a "dad joke of the day."

Chuck runs everything on a cheap Linux cloud VM spun up on Linode (his sponsor
and cloud provider of choice) for ~1 cent/hour, reinforcing his recurring
message that the best way to learn Linux is hands-on. Running time ~14 minutes.

## Key claims (paraphrase unless quoted)

- (2022-04-12) Bash is powerful, makes you a strong IT engineer, and can also make each day of your life better. [concept/motivation]
- (2022-04-12) A shell script starts with a shebang line; `echo` prints messages and `sleep` pauses between them; `chmod +x <script>` makes it executable and `./script.sh` runs it. [episode-one recap]
- (2022-04-12) A **variable** is set with `name="value"` (no spaces around `=`, value in quotes) and reused by referencing `$name`; think of a variable as something that "can change." [intro concept]
- (2022-04-12) Variables save time and reduce error — change one value and every reference updates at once, instead of editing each name by hand. [why-it-matters]
- (2022-04-12) `read name` captures interactive user input and stores it in the `name` variable — lets the script ask "what is your name?" and personalize output. [intro concept]
- (2022-04-12) **Positional parameters** (`$1`, `$2`, …) match arguments passed on the command line in order: `./best_day_ever.sh Abby ice-cream` sets `$1=Abby`, `$2=ice-cream`. Also called positional arguments/parameters. [intro concept]
- (2022-04-12) **Command substitution** — `variable=$(command)` — stores a command's output in a variable; demoed with `user=$(whoami)`, `date=$(date)`, and a variable set to `$(pwd)`. [intro concept]
- (2022-04-12) Helper commands: `whoami` shows the logged-in user, `pwd` (print working directory) shows current location, `date` shows the current date/time. [supporting commands]
- (2022-04-12) The full "Bash Butler" script additionally outputs private IP, public IP, weather for a chosen city, and a dad joke of the day. [teaser of finished result]
- (2022-04-12) Chuck uses Linode to spin up a Linux VM in the cloud for about one penny per hour; new Linode signups get $100 free credit for 60 days. Learning Linux is best done hands-on. [tooling/sponsor]
