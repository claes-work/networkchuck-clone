# Most AI coding tools don't sandbox on Windows (except one)

## Summary

Chuck flags that most AI coding agents don't sandbox on Windows — Linux has bubblewrap, macOS has seatbelt, but Windows has no equivalent, so tools "punt" (tell you to use WSL or don't sandbox at all). He highlights OpenAI as the exception: they built a Windows security sandbox from scratch in Rust (using built-in Windows features) and open-sourced it. Ends promoting the Codex app for Windows. Has a promo lean toward OpenAI's Codex but carries genuine technical framing. Domains adjusted to include cybersecurity (sandboxing) rather than cloud-devops.

## Key points (dated)

- 2026-04-24 — Linux ships bubblewrap and macOS ships seatbelt as standard built-in sandboxes; Windows has nothing comparable, so most AI coding tools either recommend WSL or skip sandboxing.
- 2026-04-24 — OpenAI built its own Windows security sandbox from scratch in Rust: restricted tokens, filesystem ACLs, dedicated sandbox users, firewall rules — all using features built into Windows — and open-sourced the whole thing.
- 2026-04-24 — Chuck says he asked the OpenAI team why no one else does this; they answered "It's kind of hard." He praises support for Windows developers and plugs the Codex app for Windows.
