# I started a blog.....in 2024 (why you should too)

## Summary

Solo video in which Chuck Keith argues that everyone — especially people in IT
careers — should start a blog in 2024, and then walks through the "insane,"
deliberately over-technical publishing pipeline he built for his own blog
(blog.networkchuck.com). The WHY is framed around Tiago Forte's *Building a Second
Brain*: Chuck has spent over a year capturing, organizing, and distilling
information, but has neglected the final step of the CODE framework — **Express**.
He's starting a blog to force himself to formulate and share his own ideas, first
"for me and for them" (his children, grandchildren, friends, and family), and
secondarily on the chance it reaches people outside his circle who can only be
reached by his singular perspective.

The HOW is an own-your-platform, notes-first workflow that keeps writing inside his
existing tools: he writes a note in **Obsidian** → a **Python** script formats/copies
images → **Hugo** (static-site generator, "terminal" theme) turns the markdown into
HTML → the site is pushed to **GitHub** → a **webhook** tells his host to pull and
deploy. His hosting provider and the video's sponsor is **Hostinger** (rendered in
the auto-captions as "hosting here / hosting your"; deal link hostinger.com/networkchuck10).
Prerequisites shown: Git and Go (Golang), SSH-key auth to GitHub, a Hugo `git subtree`
push of just the `public/` folder to a dedicated deploy branch, and Hostinger auto-deploy
via GitHub webhook. He collapses all the manual steps (robocopy/rsync sync, Python image
copy, Hugo build, git add/commit/push to both branches) into one "mega script" —
`update-blog.ps1` on Windows and a `.sh` equivalent on Mac/Linux. He closes by telling
viewers the pipeline is optional: use Hostinger's AI website builder or WordPress if they
prefer — the point is to just start writing and expressing ideas.

## Key claims (dated — WHY + how)

WHY:
- (2024-11-27) *Building a Second Brain* by Tiago Forte changed how Chuck relates to
  information: "my mind is for having ideas, not holding them," we consume too much,
  and we're "surrounded by knowledge but starving for wisdom." (paraphrase)
- (2024-11-27) The goal of a second brain is to **Express** — and Chuck admits he's been
  bad at the expressing/sharing step despite a year of capturing and organizing. A blog
  forces him to do the work of formulating his ideas. (paraphrase)
- (2024-11-27) You only truly know what you make, and you only find out what's valuable by
  sharing it; some people can only be reached by your singular perspective — "only if you
  write that blog or record that video." (paraphrase)
- (2024-11-27) He's starting the blog primarily for himself and the people he cares about
  (his six daughters, grandchildren, friends, family), not to be the go-to expert on any
  topic — and "your people want to hear from you too." (paraphrase)
- (2024-11-27) He cites Daniel Miessler's post (rendered "Daniel Mesler" in captions) that
  in an AGI-near future you should "know your life, know your goals" and "start writing even
  if you think you don't have anything to say," reinforcing the blog thesis. (paraphrase)

HOW (the pipeline / stack):
- (2024-11-27) Write posts as markdown in **Obsidian**; only a dedicated `post/` folder is
  published — the rest of the vault (including his diary) stays private. (paraphrase)
- (2024-11-27) **Hugo** (static-site generator) converts the markdown to a website; he uses
  the free **terminal** theme from themes.gohugo.io. Prereqs: **Git** and **Go/Golang**.
  On Windows he installs the Hugo binary and adds it to the PATH system environment variable.
  (paraphrase)
- (2024-11-27) Sync the Obsidian `post/` folder into Hugo's `content/post/` with **robocopy /MIR**
  on Windows or **rsync** on Mac/Linux. (paraphrase)
- (2024-11-27) A **Python 3** script (`images.py`, separate Windows and Mac/Linux versions)
  finds image links in posts, reformats them for Hugo, and copies images out of Obsidian's
  managed attachments directory into Hugo's `static/images/`. (paraphrase)
- (2024-11-27) Add YAML **front matter** (title, date, tags) to each Obsidian note; this can be
  templated with the **Templater** plugin. (paraphrase)
- (2024-11-27) Push the Hugo site to **GitHub** (authenticated with an SSH key, not password);
  keeping the repo public lets readers open pull requests / comment on his ideas "like code."
  Then use a **git subtree** to publish only the built `public/` folder to a dedicated deploy
  branch the host pulls from. (paraphrase)
- (2024-11-27) Host on **Hostinger** (the sponsor). The blog is served at a subdomain
  (blog.networkchuck.com) with DNS managed at Cloudflare via an A record pointing to the
  Hostinger server IP (proxy set to DNS-only so the SSL cert installs). Hostinger's Git
  section + a **GitHub webhook** enables auto-deploy: new commits trigger a pull and rebuild.
  (paraphrase)
- (2024-11-27) All steps are collapsed into one "mega script" — `update-blog.ps1` (PowerShell)
  on Windows, a `.sh` script on Mac/Linux — so publishing is: write in Obsidian → run one script
  → it's live. Set the source/destination path and repo variables at the top before use. (paraphrase)
- (2024-11-27) The pipeline is intentionally over-complicated; the easy alternatives he endorses
  are Hostinger's AI website builder or WordPress. The real ask is just to start. (paraphrase)
