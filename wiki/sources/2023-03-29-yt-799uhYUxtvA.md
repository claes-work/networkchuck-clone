# build your own browser (crazy SECURE)

## Summary

Chuck walks through self-hosting a "crazy secure" disposable browser using
[Kasm](../entities/kasm) (spoken as "chasm" in the auto-captions) — the same
technology behind his commercial [[../entities/networkchuck-cloud-browser]],
which he had launched two weeks earlier. The video is framed as a response to
community pushback about selling a product that people can host for free: Chuck
deliberately shows viewers how to stand up the DIY version themselves, because
his stated priority is that people browse safely regardless of whether they pay
him.

The core concept: Kasm runs a web browser inside a Docker container on a server,
then streams the browser to the user's local browser. Nothing browsed touches
the user's real machine, and when the session ends the container is deleted and
everything is gone — that disposability and isolation is what makes it secure.
The build deploys Kasm on a Linode (Akamai) cloud instance via a single install
command, then accesses the Kasm workspace UI to launch containerized browsers
(e.g. Brave). Chuck also covers the Kasm Chrome extension for "open link in
Kasm" convenience.

The second half is a candid cost/convenience/security comparison between
self-hosting and his paid Cloud Browser, plus a privacy Q&A addressing viewer
concerns about tracking, anonymity, and law enforcement.

## Key claims (dated — 2023-03-29)

- Kasm workspaces work by launching a web browser inside a Docker container on a
  server and streaming it to the user's local browser; when finished, the user
  deletes the browser and the container and it is all gone — this disposability
  is a core part of what makes it secure. (paraphrase)
- The NetworkChuck Cloud Browser is essentially built on Kasm workspaces; the
  self-hosted Kasm build shown here is the DIY equivalent of that paid product.
  (paraphrase)
- Kasm (the software) is free and can be installed and hosted anywhere — on-prem
  in a home lab or in the cloud — but the underlying hardware, internet, and
  storage are not free. (paraphrase)
- Kasm's stated minimum system requirements are two cores, 4 GB of RAM, and
  50 GB of storage (SSD ideally); this maps to at least the Linode 4 GB plan at
  roughly $20/month. (paraphrase)
- Cloud instances are billed hourly, so spinning up a test server, using it for
  an hour or two, and deleting it costs only a few cents (~3 cents/hour).
  (paraphrase)
- Build steps: create a Linode (Ubuntu 22.04, ~4 GB plan), SSH into it, run
  `sudo apt update`, then paste the single Kasm install command, which handles
  the rest of the installation automatically over a few minutes. (paraphrase)
- Access is via the server's reverse-DNS address over HTTPS; Kasm prints both a
  user and an admin login credential to the terminal during install (admin login
  is `admin@kasm.local`). (paraphrase)
- After logging into the admin Kasm UI, the user goes to Workspaces, picks a
  browser (e.g. Brave), and launches a session — which spins up a Docker
  container running that browser, deletable when done. (paraphrase)
- The self-hosted Kasm option has an official Chrome extension (search "Kasm" in
  the Chrome Web Store); after setting the Kasm URL to your own server and
  choosing a default workspace image, you can right-click any link and "open in
  Kasm" to launch it inside the self-hosted secure browser. (paraphrase)
- Self-hosting is cheaper only in specific cases and generally ends up costing
  more money and effort than the $7/month Cloud Browser, since you must support,
  update, and expose the service yourself. (paraphrase)
- Self-hosting restricts you to one server location; the paid Cloud Browser can
  launch browsers from multiple locations (Europe, India, South America, USA).
  (paraphrase)
- On privacy: activity inside the container is not seen, logged, key-logged,
  screen-recorded, or collected as telemetry; only application-level data (user
  logins, session/container requests) is tracked, mainly for performance,
  stability, and compliance as a cloud service provider running on AWS and OCI.
  (paraphrase)
- Because many users can share the same load-balanced IP address and in-container
  activity is not tracked, tracing any one person's activity back to them would
  be difficult — though the service is explicitly not built for or advertised for
  illegal activity, and US law will be obeyed (logs handed to law enforcement if
  legally required). (paraphrase)
- Chuck states Kasm had been operating as a SaaS for four years with no incidents
  with law enforcement. (paraphrase, self-reported)
- Sponsors/context: Linode (Cloud Computing by Akamai) is the sponsor; new Linode
  users get $100 credit for 60 days. (paraphrase)
