# Docker Compose will BLOW your MIND!! (a tutorial)

## Summary

A beginner-to-intermediate tutorial in which Chuck Keith teaches [[../topics/cloud-devops/cloud-devops|Docker Compose]] — the Docker tool that builds "entire dockerized worlds with one file," replacing the tedium of running containers one at a time with `docker run`. He opens with the payoff: one command spins up a 14-container Kali Linux hacking lab spanning three networks (13 vulnerable machines), and one command tears it all down.

Chuck builds up from scratch: he installs `docker.io` and `docker-compose` on Ubuntu, then contrasts the "old fashioned" `docker run` against the equivalent `docker-compose.yml`. He walks through the anatomy of a compose file — `version`, `services` (containers), `networks`, `volumes` — stressing that YAML is indentation-sensitive ("YAML loves its space… just like Python"). He demonstrates a single-container Nginx service, adds a second, creates a custom bridge network with a defined subnet via IPAM, and attaches a container to it with a static IP. The centerpiece is a two-container WordPress + MySQL deployment showing `environment` variables, `depends_on`, and a `volumes` mapping so the database survives container teardown. Prerequisites: basic Docker knowledge, some Linux (he uses Ubuntu desktop in VirtualBox), and coffee. Includes a Bitdefender sponsor read.

## Key claims (paraphrase, dated 2022-08-31)

- Docker Compose lets you build an entire dockerized environment from a single file, saving the tedium of creating and running containers line-by-line with `docker run`. A compose file is essentially the `docker run` command in a different format.
- To install on Ubuntu: `sudo apt update`, then `sudo apt install docker.io docker-compose -y`.
- Best practice: make a dedicated folder for each compose project. Compose names containers, networks, and volumes using the folder name as a prefix (e.g. folder `coffee_time` + service `website` → container `coffee_time_website_1`).
- Name the file `docker-compose.yml` — Compose looks for that filename by default; otherwise you must point to it with `-f`. Run compose commands from the same directory as the file.
- YAML (`.yml`) is a data serialization language, but for practical purposes think of it as a human- and machine-readable configuration file. It is whitespace/indentation-sensitive like Python.
- Core compose file structure: `version:` (he uses "3", the current version at recording), then `services:` (each named service defines a container), plus top-level `networks:` and `volumes:` in line with `services`.
- A minimal service defines `image:` (e.g. `nginx`), `ports:` (host:container mapping, e.g. `- "8081:80"`), and optionally `restart: always` so it comes back after a reboot.
- `docker-compose up -d` starts the stack detached (in the background); `docker-compose down` stops AND deletes the containers and networks; `docker-compose stop` merely stops containers without deleting them.
- By default Compose creates a bridge network for the stack and places containers in it. `docker-compose ps` lists only the containers in that compose environment.
- A custom network is defined under a top-level `networks:` section using `ipam` (IP Address Management), a driver (`default`, which in Docker is the bridge), and `config` specifying the `subnet`. Containers join it via a `networks:` key with an `ipv4_address` for a static IP.
- Re-running `docker-compose up -d` after editing the file only updates/recreates what changed rather than rebuilding the whole stack.
- WordPress in Docker has two components — a front-end web server and a MySQL database — which become two separate containers. He uses the `wordpress` and `mysql:5.7` images from Docker Hub.
- The `environment:` key passes container-specific variables (e.g. WordPress database host/name/password, MySQL root password and database name); WordPress's env vars must reference the MySQL service so they connect.
- `depends_on:` tells Compose to start a dependency container first (WordPress `depends_on` MySQL) regardless of file order.
- `volumes:` map a directory inside the container to a directory on the host, so data (e.g. a database at `/var/lib/mysql`) persists even if the container is deleted, restarted, or blown up — confirmed by running `docker-compose down` then `up` and finding the same WordPress site.
- Companies mainly use Docker Compose to package and deploy their apps with the right containers connected the right way; examples range from a simple Minecraft server (a volume + an image) to Pi-hole with Cloudflare DoH.
- The demo hacking lab's vulnerable containers come from VulHub, which converts vulnerable virtual machines into containers.
