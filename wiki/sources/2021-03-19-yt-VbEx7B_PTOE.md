# Linux for Hackers // EP 1 (FREE Linux course for beginners)

## Summary

The kickoff episode of NetworkChuck's "Linux for Hackers" FREE course — a third
flagship free-education series after his FREE CCNA and FREE Security+ series. Chuck
frames Linux as a required, foundational, prerequisite skill for anyone who wants to
become a hacker (and, more broadly, anyone in IT). The series is sponsored by (and built
around) Hack The Box Academy, which provides a free browser-based Linux lab that the
entire series will use; viewers are told to sign up via the description link and follow
along hands-on immediately.

After walking through the sign-up flow (Hack The Box Academy "cubes" — 30 to start, the
"Linux Fundamentals" module costs 10 cubes and refunds 10 on completion, making it
effectively free — and launching a browser-based Linux box), Chuck teaches two things:
(1) what Linux actually is, and (2) the first terminal commands. He explains that the
in-browser machine is Parrot OS, a distribution built for hacking; that Linux is
technically a kernel (not an OS) that interfaces with hardware, on top of which
operating systems like Parrot OS, Kali, Ubuntu, CentOS, and Debian are built; and why
Linux is used — open source, free to redistribute, traditionally faster and more secure,
and the base for most servers and hacking tools. He then insists the real power of Linux
is the terminal, not the GUI, and teaches four commands side-by-side with GUI equivalents:
`pwd`, `ls`, `cd`, and `cd ..`. He signs off with his usual "hack the YouTube algorithm"
call to action and a comment-section quiz.

- (2021-03-19) Hack The Box Academy uses "cubes" to unlock courses; you start with 30, the

  "basically free as long as you do it."
- (2021-03-19) Hack The Box Academy is tied to Hack The Box but is a more guided IT security
  platform; all the commands in this video are covered in depth there.

What Linux is (teaching):
- (2021-03-19) The in-browser machine is Parrot OS, a Linux distribution built for hacking;
  "OS" stands for operating system.
- (2021-03-19) An operating system (Windows, macOS, iOS, Android, Parrot OS) is the software
  we interact with to make the computer do things.
- (2021-03-19) Linux is technically NOT an operating system — Linux is a kernel; operating
  systems are built on top of the Linux kernel.
- (2021-03-19) Parrot OS is one of many distributions/flavors of Linux; others include Kali
  Linux, Ubuntu, CentOS, and Debian, all using a version of the Linux kernel.
- (2021-03-19) The kernel's job is to interface with hardware (CPU, RAM, hard drive, etc.);
  it is the middleman between the application, the OS, and the hardware (e.g., when you run a
  tool like nmap, the OS asks the kernel for hardware resources).
- (2021-03-19) Linux is open source, meaning it is open and free to use and redistribute,
  which lets people build distributions like Parrot OS.
- (2021-03-19) Linux is traditionally faster and more secure, which is why most websites and
  servers in the world run Linux, and why hackers use it — most hacking tools are Linux-based.

Terminal commands (teaching — the real point is: use the terminal, not the GUI):
- (2021-03-19) Linux has a nice GUI (graphical user interface), but it wasn't meant to be
  used that way; the true power of Linux is the terminal / command line.
- (2021-03-19) `pwd` stands for "print working directory" and shows the full path of where
  you currently are in the filesystem.
- (2021-03-19) `ls` stands for "list" and lists the contents of your current working
  directory (equivalent to viewing a folder's contents in the GUI).
- (2021-03-19) `cd` stands for "change directory"; `cd desktop` moves you into the Desktop
  folder (equivalent to double-clicking a folder).
- (2021-03-19) `cd ..` moves you back up one directory (equivalent to the GUI back button).
- (2021-03-19) Repeatedly running `cd ..` eventually lands you at `/` (forward slash), the
  root of the filesystem — "the end."
- (2021-03-19) End-of-video quiz: from root `/`, getting back to the starting directory can
  be done with either one command or two commands — answer in the comments.
