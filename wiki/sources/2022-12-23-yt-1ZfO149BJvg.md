# you NEED to use VS Code RIGHT NOW!!

## Summary

A "you need to use X right now" beginner tutorial in which Chuck Keith pitches
Visual Studio Code (VS Code) as the one tool IT pros should use "for pretty much
everything" — writing code, managing servers, hacking, and cloud work. Framed
around his signature coffee-break format, the video walks from zero (install) to
Chuck's actual daily workflow. He shows: running VS Code in the browser (hit `.`
on any GitHub repo), local install from code.visualstudio.com, and spinning up a
VS Code *server* in the cloud on Linode (the sponsor) so you can code from
anywhere, even a phone. He then covers core UI (folders-as-projects, command
palette, tabs, integrated terminal, file explorer, search) and the color/syntax
highlighting that helps you "code better."

The heart of the video is **extensions**, which he calls the real source of VS
Code's power. He installs the Microsoft Python extension (unlocking linting,
IntelliSense autocomplete, and a run button), then his self-declared favorite
feature: **Remote - SSH**, which lets him open any remote machine's filesystem
and terminal inside VS Code — how he manages all his Linode/servers. On the
remote host he demonstrates that extensions must be reinstalled per host, previews
images/videos on a headless Linux box, manages Docker containers (view images,
inspect, attach a shell, open mapped ports in the browser) via the Docker
extension, and manages cloud resources directly through the Azure/AWS extensions.
Closes with the standard "hack the YouTube algorithm... ethically" call to action.

## Key claims (dated — concepts)

All dated 2022-12-23, paraphrased unless quoted.

- VS Code is, per Chuck, the best way to write code, manage servers, do hacking,
  and cloud work — the tool "the pros" use for pretty much everything.
- VS Code is "literally everywhere": pressing the period key inside any GitHub
  repository opens that repo in a browser-based VS Code instance.
- The best way to use it is to download and install locally from
  code.visualstudio.com; it installs like any normal application on any OS.
- You can run a **VS Code server in the cloud** (demoed via the Linode
  Marketplace) and reach it from any device, including a phone, via the server's
  reverse-DNS entry.
- VS Code treats an opened **folder as a project**; all your project files live
  in that folder. It prompts you to trust the folder's author before opening.
- The **command palette** is a central control surface you invoke constantly.
- **Syntax/color highlighting** (functions, strings, matched parentheses) helps
  you code better and track which context you're in.
- **Extensions are the most powerful feature** of VS Code — the real source of
  its power.
- The **Python extension** (from Microsoft) adds linting (the squiggly-line
  warning that flags likely-wrong code), **IntelliSense** autocomplete ("your
  little buddy helping you code"), and a run/play button.
- Windows lacks Python by default (Linux/Mac ship it); VS Code will prompt you to
  select an interpreter and install Python from the Microsoft Store.
- VS Code has an **integrated terminal** (defaults to PowerShell on Windows) that
  can be split into multiple panes — you run code and terminal commands without
  leaving the editor.
- The explorer also surfaces open editors, a code outline, a timeline, and a
  project-wide **search** across every file (plus per-file Ctrl+F find/replace).
- **Remote - SSH** is Chuck's favorite feature and his primary use case: it opens
  any remote machine's folders/filesystem over SSH, sets up a VS Code environment
  on that host, and is how he manages all his Linode and other servers.
- Extensions are **per-host** — an extension installed on your Windows machine is
  not present in a Remote-SSH environment and must be reinstalled on that host.
- VS Code can **preview images and videos** even on a headless Linux server with
  no GUI.
- The **Docker extension** (official, Microsoft) lets you view running containers
  and images, inspect a container, connect to a registry, stop/restart, attach a
  shell into the container, and open a container's mapped port in the browser.
- The **Azure and AWS extensions** let you manage cloud resources (e.g., view/create
  virtual machines) directly from VS Code without leaving the editor.
- Signature framing: this is "what the pros use," and you should start using it
  "right now."
