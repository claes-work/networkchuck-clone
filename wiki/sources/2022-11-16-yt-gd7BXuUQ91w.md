# 60 Linux Commands you NEED to know (in 10 minutes)

## Summary

A rapid-fire, beginner-oriented reference video in which Chuck Keith runs through
roughly 60 essential Linux terminal commands in about ten minutes. He works live in a
Linux VM (spun up on Linode, the video's sponsor) and demonstrates each command by
actually running it, grouping them loosely by function: remote access, filesystem
navigation, file creation/editing/viewing, file operations, links, user/permission
management, package installation, help/documentation, downloading, archiving, text
processing, networking, firewall, system info, process management, and services. The
tone is fast and playful (coffee-break asides, jokes about "man finger"), aimed at
newcomers who want a one-pass survey of what the shell can do rather than deep
coverage of any single command.

## Key claims (paraphrase, dated 2022-11-16)

- **Remote access:** `ssh user@server` connects to a Linux machine; you can't use
  Linux unless you can reach it first.
- **Navigation:** `ls` lists files (`-l` for a long list, `-a` to show hidden files);
  `pwd` prints the working directory; `cd` changes directory (`cd ..` goes up one
  level, `cd` with nothing returns home, or specify a path).
- **Creating files:** `touch <name>` is the quickest way to create a file (can create
  several at once; `-d` lets you set a creation date, even in the future).
- **Writing to files / editing:** `echo` prints text or, with `>`, redirects output
  into a file; `nano <file>` is the beginner-friendly editor (save with Ctrl+X, Y,
  Enter); `vim <file>` is the "pro" editor (`i` to insert, `Esc` then `:wq` to write
  and quit).
- **Viewing files:** `cat <file>` dumps a file's contents quickly; `less` shows one
  page at a time; `head` shows the beginning, `tail` shows the end.
- **Destroying content:** `shred <file>` overwrites a file so its contents can't be
  recovered.
- **File operations:** `mkdir` makes a directory; `cp <file> <dest>` copies; `mv`
  moves (or renames); `rm` removes a file; `rmdir` removes an empty directory; `rm -r`
  removes a directory recursively.
- **Links:** `ln -s <file> <link>` creates a soft (symbolic) link.
- **Terminal housekeeping:** `clear` cleans the terminal screen.
- **Users & identity:** `whoami` reports the current user; `adduser`/`useradd` creates
  a user; `su <user>` switches user; `exit` leaves the session/switched user;
  `passwd <user>` sets or changes a password.
- **Privilege:** `sudo` runs a command with elevated (root) permission when your user
  lacks it.
- **Inspecting users:** `finger <user>` inspects another user (must be installed
  first).
- **Package management:** on Debian/Ubuntu use `apt` (run `sudo apt update` before
  installing, then `sudo apt install <package>`); Red Hat/CentOS use `yum`; Arch and
  others are "on your own."
- **Help / documentation:** `man <command>` opens the manual (Q to quit); `whatis`
  gives a quick one-liner; `which` shows one location of a binary; `whereis` shows all
  locations.
- **Downloading:** `wget <url>` downloads files from the internet; `curl <url>` is
  Chuck's personal favorite (redirect output with `>` into a file).
- **Archiving:** `zip <archive> <file>` compresses; `unzip <file>` extracts.
- **Text processing / comparison:** `cmp` compares two files and tells you *where* they
  differ; `diff` tells you *what* differs; `sort` sorts alphabetically (demonstrated by
  piping `cat bible | sort`).
- **Finding files:** `find <dir> -name <pattern>` searches by name (supports regex,
  hidden files, empty directories, executables).
- **Permissions & ownership:** `chmod +x <file>` makes a file executable; `chown
  <user> <file>` changes ownership.
- **Networking basics:** `ifconfig` shows IP info (may need installing); `ip address`
  is the more modern equivalent; `grep` filters output (e.g. `ip address | grep eth0`);
  `awk` extracts specific fields with a regex.
- **DNS:** `cat /etc/resolv.conf` shows DNS config, or on newer Ubuntu use
  `resolvectl status`.
- **Connectivity / routing:** `ping <host>` checks if something is up (`-c` limits the
  count, packet size configurable, Ctrl+C to stop); `traceroute <host>` shows each hop
  and its latency along the path.
- **Ports & sockets:** `netstat` shows network statistics (`netstat -tulpn` is more
  useful); `ss -tulpn` is the modern equivalent.
- **Firewall:** raw `iptables` is complex, so `ufw` (which sits on top of iptables) is
  easier — `sudo ufw allow 80`, `ufw status`, `ufw enable`.
- **System info:** `uname` (with `-a` for full detail) reports system info; `neofetch`
  is a prettier version (install first); `cal` shows a calendar; `bc` does math (echo a
  problem to `bc`).
- **Resources:** `free` shows memory and swap usage; `df` shows disk space (`df -H` is
  the one you'll use all the time).
- **Processes:** `ps` lists processes (`ps aux` for the full view); `top` shows live
  resource-hungry processes; `htop` is the prettier version.
- **Killing processes:** `kill -9 <PID>` forcibly kills a process by ID (find the PID
  with `ps` + `grep`); `pkill -f <name>` kills by name without needing the PID.
- **Services / daemons:** `systemctl` starts/stops/restarts services on systemd
  systems (`systemctl stop|start|restart|status apache2`); older systems use `service`.
- **History & power:** `history` shows all commands run; `sudo reboot` reboots; `sudo
  shutdown` powers off (about one minute; `-h now` to do it immediately).
