# 3 Cisco CLI (Command-line) Hacks (CCNA)

## Summary
Chuck Keith demonstrates three Cisco IOS command-line tricks he uses daily as a network engineer to work faster in the CLI. The three hacks are: (1) the `include` / `exclude` pipe switches to filter `show` command output, (2) the `section` pipe switch to extract just the relevant block from a running config, and (3) the `do` command to run `show`/exec commands without leaving configuration mode. He frames these as time-savers he wishes he'd had when starting out, and points viewers to a free David Bombal Udemy course covering more CLI hacks. (The video's tail is a CBT Nuggets giveaway winner announcement, which is out of scope for this technical page.)

## Key claims
- (2017-11-06) Hack 1 — the `include` and `exclude` pipe switches filter `show` command output. `show run | include SNMP` (or `show run | include clock`) shows only config lines containing the keyword; Chuck uses `include` mainly to quickly check whether a specific command/route/config line is present.
- (2017-11-06) `exclude` removes matching lines from output. Example: `show processes CPU | exclude 0.00` hides idle processes so only processes actually consuming CPU remain — useful when investigating a high-CPU alert to find the offending process.
- (2017-11-06) Another `exclude` use: `show mac address-table | exclude SCP` strips all the phones (SCP entries) from the MAC address table so you can find a non-phone host; conversely `include` a partial MAC (e.g. `c471`) to find a specific address quickly.
- (2017-11-06) Hack 2 — the `section` pipe switch (used with `show run`) returns the full running-config block for a matching keyword. `show run | section router` shows the entire routing configuration entered via CLI; `show run | section ospf` narrows it to just the OSPF config. Chuck calls this his fastest way to see exactly which commands were entered to configure a protocol like OSPF.
- (2017-11-06) Hack 3 — the `do` command lets you run exec/`show` commands from inside any configuration mode. While configuring an interface (e.g. `interface vlan 9999`), `do show ip interface brief` runs the show command without exiting config mode. Chuck says every engineer he has worked with uses this constantly.
- (2017-11-06) He recommends David Bombal's free Udemy course on CLI hacks as further learning (and notes Bombal has strong GNS3 content).
- (2017-11-06) Aside: as of ~2017 the network engineer's day-to-day is "starting to change a little bit" — a nod to network automation/Python (he references his own "CCNA or Python" video).
