# Claude Code on your Phone is OFFICIAL (it changes everything)

## Summary

A short reaction video Chuck films while traveling in Japan with his family, covering
Anthropic's official feature that lets you control a running Claude Code session from
your phone. Chuck refers to it as "Claude Code Remote" or "remote control" (he admits
he forgets the exact name). The workflow: with an active Claude Code session in the
laptop terminal, you type the `/remote-control` slash command (or append the
`--remote-control` option after the `claude` command); this exposes the session to the
native Claude mobile app, where you can resume it, continue chatting, and see your
input mirrored back on the laptop terminal in real time. It also generates a QR code for
connecting from the phone.

Chuck's core point ("changes everything") is that this is NOT the same as his earlier
DIY setup — a remote terminal on your phone to reach Claude Code from anywhere. This is
handled entirely by Anthropic: no VPN, no remoting into a machine yourself, no
self-hosted tooling. Crucially, the phone session runs on your laptop's (or remote
server's) resources, so all your configured skills and project context are available —
he demos triggering a "make a thumbnail" skill from his phone while it executes on his
laptop. He frames it as a threat to OpenClaw (his term, referring to what he'd been
using such tools for), since it elegantly solves the "access my Claude Code CLI while
I'm out" problem. He notes it's a research preview, limited to Mac and (he thinks) Pro
or Max users. He also touches the `ask` tool surfacing questions in the app, and
notification support that wasn't fully working for him yet (thumbnails/files didn't
actually transfer to the phone). His one feature request: the ability to start a brand
new session from the phone (currently you must start it on the laptop first). Video ends
with a Network Chuck Academy plug (CCNA course by Jeremy Cioara) and his signature
closing prayer.
