# Do you STILL suck at subnetting?? (THE FINAL TEST) // EP 9

## Summary

The ninth and final episode of NetworkChuck's "You SUCK at Subnetting" series is the capstone challenge that wraps the whole run. Chuck poses one scenario: Bernard Hackwell, "CEO of Hackwell Industries," needs a new factory addressed with four networks — guest, robots, servers, and workers — each with different host requirements, and all the addressing space available is a single 172.21.42.0 /24 network. The solution is VLSM (Variable Length Subnet Mask): subnetting a subnet into pieces of different sizes.

Chuck frames this as the series finale and the student's "final challenge." His core teaching point is that VLSM is not a new skill — it is exactly the host-based subnetting taught earlier in the series, just applied repeatedly and approached in a specific order: work from the largest host requirement down to the smallest, and each new subnet picks up where the previous one left off. He works the four networks in order (workers → robots → servers → guest), reusing his recurring "Nosferatu / Nosfera2" binary-conversion characters and the "upside down / save the host" method from prior episodes, speeding through the mechanics because the audience has "already been here before."

The result: from one /24, he carves a /25 (workers), /26 (robots), /27 (servers), and /28 (guest). Chuck closes by declaring the viewer "no longer sucks at subnetting," warns them they will forget it and need practice (plugging Network Chuck Academy), and confirms the subnetting series is over — though more CCNA content is coming. Boson software is credited as the series sponsor.

## Key claims (dated — the challenge/method)

All dated 2023-08-23 (paraphrase unless quoted):

- The challenge: given a single 172.21.42.0 /24 network, address a new factory needing four separate networks (guest, robots, servers, workers), each with its own differing host requirement.
- VLSM (Variable Length Subnet Mask) is the tool — "variable" meaning the subnets being cut up can be different sizes.
- Feasibility check: the four networks total roughly 210 hosts; a /24 accommodates 253 hosts, so the scenario is possible but tight.
- Method rule 1 — order matters: when doing VLSM, always start with the largest host requirement first and work from biggest to smallest.
- Method rule 2 — continuity: each new subnet starts where the previous one left off (you "pick up where you left off"), reusing the same host-based subnetting process taught throughout the series.
- Workers (117 hosts): 7 host bits needed → /25 subnet mask (255.255.255.128), increment 128; network 172.21.42.0 through 172.21.42.127.
- Robots (57 hosts): → /26 subnet mask (255.255.255.192), increment 64; network starts at 172.21.42.128 (128 + 64 = 192 boundary).
- Servers (26 hosts): → /27 subnet mask (255.255.255.224), increment 32; network starts at 192, ending around .223.
- Guest (host requirement in the low-20s): → /28 subnet mask (255.255.255.240), starting at 224, ending around .239.
- Overall result: one /24 was sliced into /25, /26, /27, and /28 subnets of different sizes.
- Learning framing: VLSM is "a lot of what we've already learned" — subnetting based on host requirements four times, approached big-to-small.
- Retention caveat: subnetting is a skill that must be developed over time; you will forget it and need to relearn/practice (Chuck says it happens to him). Practice offered via Network Chuck Academy.
- Transferability: subnetting is a core skill carried into other networking problems (e.g., diagnosing why an access list isn't working), not just standalone "please subnet this" questions.
- Series status: this is the finale of the subnetting series; more CCNA content is still to come, but the subnetting portion is over.
- Sponsor: Boson software (Boson ExSim practice exams and Boson NetSim network simulator) is credited as making the free CCNA/subnetting series possible.
