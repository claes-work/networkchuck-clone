# How Hackers hide files on Windows 10 and Linux

## Summary

An ethical-hacking / security-awareness walkthrough of two file-hiding techniques, framed around the attacker's goal of "covering your trail" and the defender's job of detecting the hidden data. Chuck demonstrates (1) hiding a file inside another file on **Windows** using **NTFS Alternate Data Streams (ADS)** — a file-system feature, not a bug — and (2) hiding a file inside a JPEG on **Kali Linux** using **steganography** with the `steghide` tool. He also shows the simpler `attrib +h` hidden-file attribute as a contrast.

Throughout, Chuck pairs each attack with its detection/forensics counterpart: ADS streams are revealed with `dir /r`; the hidden attribute is defeated by enabling "show hidden files" in Explorer; and he stresses that on modern systems these techniques are trivially caught by scanners and forensic tools. The framing is educational (CEH exam prep, awareness) with an ethical-use disclaimer, not an operational evasion guide. Sponsored by ITProTV.

## Key claims (dated — techniques + defense)

All dated 2020-09-14 (video publication); all paraphrase unless quoted below.

**Techniques**
- Staying hidden and covering your tracks is a core part of ethically hacking a target machine — the motivation for learning file-hiding techniques.
- **Alternate Data Streams (ADS)** is a feature/file attribute of the **NTFS** file system (what Windows uses) that lets you legitimately hide one file's contents inside another file. Chuck says he "had no idea this was a thing."
- ADS is performed from the command prompt (CLI) with a `type` redirect into a stream named with a colon on the host file, e.g. `type secret.txt > host.txt:secret.txt`; the secret's stream rides inside the visible host file.
- After hiding, the original standalone secret file is deleted (`del`), so `dir` and File Explorer show only the innocent host file; the hidden content is still retrievable by opening the named stream (e.g. `notepad host.txt:secret.txt`).
- The simpler **hidden attribute** technique uses `attrib +h <file>`, which removes the file from a normal `dir` listing and Explorer view.
- **Steganography** hides a file inside an image "in plain sight" — Chuck calls it "movie secret agent stuff." Demonstrated on Kali Linux with the `steghide` tool (installed via `sudo apt install steghide -y`).
- `steghide` usage shown: `steghide embed -cf <cover.jpg> -ef <secret.txt>`, prompting for a password; the secret is then removed from disk so only the image remains. Extraction: `steghide extract -sf <cover.jpg>` with the password recovers the file.

**Defense / forensics**
- ADS-hidden streams are discovered with **`dir /r`**, which lists alternate data streams alongside the visible files — an admin can detect the hidden stream this way.
- The `attrib +h` hidden attribute is trivially bypassed: Explorer → View → Options → change folder/search options → View → "show hidden files, folders and drives"; Chuck (as an admin) always keeps this enabled, so such files were never hidden from him.
- None of these methods are foolproof. On modern systems ADS "won't work" for seriously nefarious purposes — during a real attack, pen test, or forensic investigation, scanners and forensic tools "will find these files without even blinking."
- The techniques are still worth learning because they appear on the **CEH** exam (knowing what ADS is in NTFS and how to hide files in Windows) and because steganography has legitimate, usable use cases.
- Ethical-use disclaimer: only hack systems you have permission to test.
