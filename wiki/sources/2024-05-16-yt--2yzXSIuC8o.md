# monitor all your stuff RIGHT NOW!!

## Summary

Chuck walks through deploying a full IT monitoring system for a home lab (or
enterprise) so you can answer two questions about every device: "is it up?" and,
more importantly, "is it actually working?". The tool used and sponsored is
**Progress WhatsUp Gold** (abbreviated "WG" throughout), which had just released a
free edition for home labs that monitors up to 10 devices. Chuck notes it was the
first IT monitoring solution he ever used, back on the help desk.

The bulk of the video is a conceptual + hands-on tour of the monitoring
fundamentals that underpin any such tool, not just WhatsUp Gold: ICMP ping for
liveness, and the two data-collection protocols **SNMP** (cross-platform) and
**WMI** (Windows). He explains OIDs, MIBs, community strings, SNMP polling vs.
traps, and the security tradeoffs of SNMP versions. He then installs WhatsUp Gold,
runs a network discovery scan that auto-detects device roles and builds network
maps, monitors devices, and demonstrates dashboards, storage monitoring, alerting
(a Slack webhook alert on a downed service), and two custom monitors: an HTTP/TCP
port monitor for a self-hosted Open WebUI server and a **custom SSH monitor** to
poll GPU temperature on his AI server. Closes by pitching monitoring as a career /
resume-building skill.

## Key claims (dated — the tool + setup)

All from 2024-05-16, paraphrased unless quoted:

- The monitoring tool used is **Progress WhatsUp Gold** (the sponsor), which had
  just released a **free edition for home labs** that monitors **up to 10 devices**
  — and, unlike sensor-limited competitors, all metrics on those 10 devices count
  as one device each rather than per-sensor.
- The most basic monitoring check is liveness via **ICMP echo request/reply** (ping);
  if replies stop, the system marks the device down and alerts.
- **SNMP** (Simple Network Management Protocol) lets a monitoring system connect to a
  device and collect stats (CPU, memory, network, disk) via **OIDs (object
  identifiers)** — long numeric strings; generic OIDs work across most devices, while
  vendors like Cisco ship proprietary OIDs bundled in **MIBs (Management Information
  Bases)** that you download and upload into the monitoring system.
- SNMP normally works by **polling** (roughly every 10 minutes by default, over
  **UDP port 161**); for time-critical metrics like temperature you can shorten the
  poll interval or configure an **SNMP trap** so the device pushes an alert on a
  threshold instead of waiting to be polled.
- Setup demo (Linux): install with `sudo apt update` then `sudo apt install snmpd`;
  verify it's listening on port 161 with `ss -tuln`; edit `/etc/snmp/snmpd.conf`,
  set `agentaddress` to `udp:161` (all interfaces) and configure the **community
  string** (a password, sent in **plaintext** on v1/v2c). Default community string on
  many devices is `public`; Chuck changes it to `networkchuck`.
- SNMP can be **read-only or read-write**; a read-write community string lets the
  monitoring system change device settings (he demos changing a hostname), which is
  dangerous in an attacker's hands. He reverts to read-only.
- **SNMPv3** adds encryption plus username/password auth and view restrictions and is
  the production-grade choice, but the demo stays on **v2c** for maximum device
  compatibility (many devices don't support v3).
- Servers, printers, and even Macs ship with SNMP; configure the same community
  string (or v3 credentials) across all devices. Optionally set up **SSH** with
  consistent credentials so WhatsUp Gold can log in and run custom commands.
- **WMI (Windows Management Instrumentation)** is the Windows equivalent, installed
  and running by default, but the Windows firewall usually blocks it and ICMP — allow
  both via PowerShell rather than disabling the firewall. WMI **requires a Windows
  user account with administrative privileges** (managed via `lusrmgr.msc`).
- WhatsUp Gold system requirements: officially a Windows Server (works unofficially on
  Windows 11, including a VM); **quad-core CPU, 8 GB RAM minimum, 25 GB storage**. The
  installer bundles **SQL Server Express** as the database and binds **IIS to port 443**
  with a self-signed certificate.
- Workflow: run a **discovery scan** with SNMP/WMI/SSH credentials → WG auto-detects
  device names, roles, and monitoring capabilities and can auto-build **network maps**
  → select up to 10 devices to actively monitor → view dashboards (up/down summary,
  top-10 CPU/disk/memory/interface over time, interface errors, storage monitoring
  from his Synology NAS, a NOC viewer slideshow).
- Custom monitoring demos: an **HTTP monitor on TCP 3000** to watch a self-hosted
  Open WebUI (local ChatGPT for his daughters); and an **SSH performance monitor** that
  runs a shell command on the AI server ("Terry") to poll **GPU temperature** — showing
  you can monitor "literally anything" reachable via CLI or PowerShell.
- Alerting: create an action (Slack webhook, PowerShell, SSH, ServiceNow, IFTTT, etc.),
  then an **action policy** defining what "down" means (e.g., alert immediately, with
  optional blackout schedules) and attach it to a device or a single monitor. He demos
  a Slack down/up alert on the Open WebUI service.
- His upgraded (licensed) WhatsUp Gold adds **configuration management** (backing up
  Cisco/Juniper switch configs), **log management** (Windows event logs + Linux syslog),
  and **network traffic analysis** via **NetFlow**, plus virtual monitoring for
  VMware/Hyper-V (he wishes it supported Proxmox).
