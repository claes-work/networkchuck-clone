# CPU, GPU…..DPU?

## Summary

An Nvidia-sponsored explainer in which Chuck introduces the DPU (Data Processing
Unit) as a third class of processor alongside the CPU and GPU. He frames the DPU
through the history of data-center virtualization: first physical servers were
consolidated into virtual machines (via VMware), then networking/security appliances
(switches, routers, firewalls, IDS/IPS) were virtualized via software-defined
networking (VMware NSX), then storage was virtualized (software-defined storage). Each
wave piled more specialized work — encryption/decryption, traffic inspection, data
movement — onto the general-purpose CPU, which was never built for it and became the
bottleneck as networks scaled from 1 Gbps to 10/100/200 Gbps.

Chuck walks up the offload ladder: a regular NIC → a SmartNIC (purpose-built card in
a PCIe slot that offloads networking/security from the CPU) → the DPU, which he
describes as "a whole body," essentially a server inside a server. He demos Nvidia's
BlueField-3 DPU, a system-on-a-chip (SoC) with its own ARM CPU, more RAM than a gaming
PC, and up to 400 Gbps throughput, programmable via Nvidia's DOCA SDK. In VMware
vSphere 8's Distributed Services Engine (formerly Project Monterey), ESXi is actually
installed onto the BlueField DPU, so the DPU runs its own OS and fully handles the
networking/firewall/storage workloads rather than merely assisting the CPU.

He runs a hands-on lab in Nvidia's data center on a 25 GbE cluster: with UPT (Uniform
Pass-Through) the DPU can be assigned as a VM's NIC and still survive vMotion (which
breaks under traditional PCIe passthrough). A Redis data-transfer benchmark shows the
DPU beating a standard CPU-bound NIC (90 vs 64 Mbps bandwidth, lower latency, ~770k vs
~548k operations/sec) while leaving the CPU unstressed. Chuck's argument for the DPU
over just adding more servers: adding servers recreates the original sprawl/power/space
problem, whereas an ARM-based DPU adds specialized, power-efficient horsepower to a
single server — he cites ~30% efficiency increases and calls DPUs "the future of
virtual machines."

## Key claims (dated — 2023-05-05, paraphrase)

- A DPU (Data Processing Unit) is a third processor class alongside the CPU (general
  purpose) and GPU (graphics/AI); it specializes in data processing — moving,
  encrypting/decrypting, and inspecting network traffic.
- The need for DPUs arises from successive waves of data-center virtualization:
  physical servers → VMs (VMware), then networking/security appliances → software-defined
  networking (VMware NSX), then storage → software-defined storage. Each wave offloads
  more onto the CPU.
- The CPU is general-purpose and not built for specialty networking/security tasks
  (encryption, high-volume data transfer, traffic analysis); as network speeds rose
  from 1 Gbps to 10/100/200 Gbps it became the bottleneck.
- A SmartNIC is a purpose-built network interface card that pops into a server's PCIe
  slot and offloads CPU-unfriendly work (traffic inspection, IDS/IPS, cybersecurity).
  It is the intermediate step before the DPU.
- The GPU is not designed for data traffic forwarding — it handles graphics and AI —
  so it does not help with the networking offload problem.
- The DPU does roughly the same offloading job as a SmartNIC but is "a whole body, not
  just an arm": a system-on-a-chip (SoC) that is effectively a server inside a server,
  its own endpoint device with its own OS.
- Nvidia's BlueField-3 is the featured DPU: an SoC built on an ARM CPU (ARM is
  power-efficient and suited to high-performance data processing), with more RAM than a
  gaming PC and up to 400 Gbps. It is typically delivered as a "DPU-enabled SmartNIC."
- The DPU is programmable via Nvidia's DOCA SDK, enabling extensive customization
  (e.g., VXLAN acceleration, Layer 4 firewall connection tracking).
- In VMware vSphere 8 with the Distributed Services Engine (formerly Project Monterey),
  ESXi is installed onto the BlueField DPU so it runs its own OS and fully executes
  networking/security/storage workloads instead of merely assisting the CPU.
- UPT (Uniform Pass-Through) in vSphere 8 lets a DPU be assigned as a VM's NIC while
  still supporting vMotion; traditional PCIe passthrough breaks vMotion (connection to
  the SmartNIC is lost).
- Lab benchmark (25 GbE cluster, Redis servers): DPU vs standard CPU-bound NIC — DPU
  reached ~90 Mbps bandwidth, ~0.19 ms latency, ~770,000 ops/sec vs the standard NIC's
  ~64 Mbps, ~0.27 ms, ~548,000 ops/sec — while the CPU stayed unstressed.
- Rationale for DPU over more servers: adding servers recreates the original
  space/power/inefficiency problem; an ARM-based DPU adds power-efficient specialized
  horsepower to one server, yielding claimed ~30% efficiency increases.
- Chuck frames DPUs as "the future of virtual machines" and expects specialized DPUs
  for advanced workloads like AI.
