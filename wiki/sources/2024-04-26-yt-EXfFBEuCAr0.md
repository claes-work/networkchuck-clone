# you STILL need a website RIGHT NOW!! (yes, even in 2025)

## Summary

An updated version of Chuck's "make your own website" career tutorial, framed around a
poll where he claims 81% of viewers still had no website after his previous video. He
argues that anyone in IT/tech should have a personal website as a home base for sharing
what they learn, and walks through the full build twice: first a free, dead-simple
version, then a fancy AI-built version. He uses his daughters (Chloe and a sister) as a
running bit — giving them AI website-building access and a $100 competition judged by
viewer vote — to prove no coding is required.

The technical how-to has two paths. **Free path:** a website is just an `index.html`
file; to get it on the internet you put that file on a web server (he explains the
client/browser sending an HTTP GET, the server responding with 200 OK). He recommends
**GitHub Pages** as a free web server — create a public repo ("my website"), upload the
`index.html` file, commit, then Settings → Pages → deploy from branch → change from
"none" to "main" → save. That publishes a live URL in about five minutes for free. He
cites network engineer **Nick Russo's** plain-text site as proof a simple site is
enough. **Custom-domain path:** buy a domain from a registrar (sponsor Hostinger,
hostinger.com/chuck, coupon "networkchuck"), then in Hostinger's DNS settings create an
**A record** pointing to a GitHub Pages IP address and delete the duplicate A record;
back in GitHub Pages, enter the custom domain, which auto-provisions an HTTPS
certificate. **Fancy path:** use Hostinger's AI website builder / managed WordPress to
generate a whole site (design, blog posts, content) from a text prompt — his argument
being that AI lets you focus on content instead of falling into the trap of endlessly
building the site.

The back half is the career rationale — four reasons you need a website (teaching
accelerates learning; your unique perspective has value; websites help you get hired;
serendipity), plus a rebuttal to the "I already have LinkedIn/YouTube/Twitter/GitHub"
objection: those platforms are someone else's, algorithm-driven, and impermanent, while
your own site is controlled, timeless, and your "home base."

## Key claims (dated — the how + rationale)

- **[2024-04-26]** Claim/motivation: a poll after his prior video showed **81% of
  viewers still had no website**; he considers the excuses (can't code, don't need it)
  invalid. (paraphrase)
- **[2024-04-26]** A website does not have to be fancy — it can be as simple as plain
  text; he cites network engineer **Nick Russo's** plain-text site as an example that
  genuinely helps a ton of people. (paraphrase)
- **[2024-04-26]** How the web works (teaching frame): a website is just a file on a
  computer; to publish it you put that file on a **web server** (another computer
  designed to serve sites). A browser (the client) sends an **HTTP GET** request and the
  server responds with a **200 OK** plus the website file. (paraphrase)
- **[2024-04-26]** Free hosting method: **GitHub Pages** is a free web server. Steps —
  sign up at github.com, create a **public repository** (e.g. "my website"), use "upload
  an existing file" to add your `index.html`, and click **Commit changes**. (paraphrase)
- **[2024-04-26]** Publish step: in the repo go to **Settings → Pages**, choose **deploy
  from a branch**, change the dropdown from **none to main**, click save, refresh — the
  site is now live at a GitHub URL. Takes about five minutes and is free. (paraphrase)
- **[2024-04-26]** Custom domain: the default GitHub URL contains "github" and your
  username; for a real domain you go to a **domain registrar** to register one — this is
  the only step that costs money, and not much. (paraphrase)
- **[2024-04-26]** DNS how-to: in the registrar's (Hostinger's) DNS/name-server
  settings, create an **A record** whose "points to" value is one of **GitHub's IP
  addresses** for GitHub Pages, then **delete the duplicate A record** with the same
  name. (paraphrase)
- **[2024-04-26]** Finishing the custom domain: back in GitHub Pages, enter the custom
  domain and save; GitHub verifies DNS and **auto-provisions a secure (HTTPS)
  certificate**. (paraphrase)
- **[2024-04-26]** Fancy/AI path: Hostinger's **AI website builder** or **managed
  WordPress** can generate an entire site — design, blog posts, content — from a typed
  prompt, no coding and no design skill required. (paraphrase)
- **[2024-04-26]** Aside/recommendation: you *should* still learn to code and learn
  **Python** ("it's good for you"), but you don't have to in order to build a website in
  2024. (paraphrase)
- **[2024-04-26]** Rationale on AI hosting: the value of a managed AI builder is it lets
  you **focus on content, not on building the site** — Chuck admits he personally falls
  into the trap of endlessly adding features and fixing breakage, ending up with a cool
  site and no content, when the whole point was to share ideas. (paraphrase)
- **[2024-04-26]** Career reason #1 — **teaching accelerates learning**: learn every new
  thing as if you're about to teach/present/blog it; shift your mindset from *taking
  notes* to *giving notes* (quoting Tiago Forte, "When one person teaches, two people
  learn," and the Einstein/Feynman idea that if you can't explain it simply you don't
  understand it). (paraphrase)
- **[2024-04-26]** Career reason #2 — **your unique perspective is valuable**; it only
  needs to reach one person to be worth it. Chuck notes his own path (network engineer →
  YouTuber) was sparked by consuming someone else's blogs/videos. (paraphrase)
- **[2024-04-26]** Career reason #3 — **a website helps you get a job**: "websites are
  the new resumes." When hiring, Chuck Googles candidates and looks at their site to see
  their writing/videos, technical diagrams, and **labs** done — evidence of effort and
  consistency. (paraphrase)
- **[2024-04-26]** Career reason #4 — a website is a **"serendipity vehicle"** (credited
  to David Perell): putting yourself out there creates unpredictable opportunities.
  Example: Mr. Beast's team found his content/site, researched him, and he got to help on
  a video — he wasn't looking for it. (paraphrase)
- **[2024-04-26]** Rebuttal to "I already have LinkedIn/X/YouTube/GitHub": those are
  **someone else's platform** — you're at their mercy (they can die, change the
  algorithm, or deplatform you); your content is a **slave to the algorithm** and gets
  lost. Your own website is **controlled, timeless, and your home base**. Still use the
  social platforms — to promote and link back to your website. (paraphrase)
