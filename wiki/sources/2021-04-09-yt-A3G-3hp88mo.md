## Key claims (2021-04-09, paraphrase)

- **Everything in Linux is a file** тАФ configuration (e.g. network settings, IP addresses, interfaces), devices (hard drives, printers, CD-ROMs), and even commands are all represented as files.
- The file system starts at the **root, `/`**; repeatedly running `cd ..` eventually lands you at `/`, where you can go no further back.
- **`/bin`** ("bin" = binaries) holds the essential command binaries тАФ the actual files that are executed when you type commands like `ls`, `cp`, `rm`, `cat`.
- A command is literally a file: you can `cat` the `ls` binary to dump its (unreadable) contents, `cp` it to a new name, `rm` it, and the copy still behaves as `ls`.
- **`/sbin`** ("super bin") holds special system-administration command binaries that only administrators use to administer the system (e.g. `adduser`).
- **`sudo`** lets you run a single command as the root/admin user when you otherwise lack permission (a "permission denied" error is fixed by prefixing `sudo`); it is covered more fully later in the series.
- **`/usr`** (the "user" directory) also contains its own `bin` and `sbin`, which mirror the root-level ones and typically hold more commands; the historical reason for the duplication comes down to hard-drive space (Chuck says look it up rather than explaining it).
- **`which <command>`** reveals where a command binary lives on disk (e.g. `which ls` returns `/usr/bin/ls`).
- **`/usr/local`** is where you should store command binaries you create yourself; `/usr` also holds shared libraries used by the command binaries, plus games.
- **`/boot`** holds the files the system needs to boot.
- **`/var`** holds things like log files and web-application-related files.
- **`/tmp`** holds temporary files that go away after a reboot.
- **`/lib`** holds shared library files, specifically things the system needs to boot.
- **`/home`** is where every regular user lives; each user has their own directory under it.
- **`/root`** is the root user's home directory тАФ separate from `/home`; listing it requires `sudo` (`sudo ls /root`).
- **`/dev`** ("dev" = devices) is where devices live as files; hard drives appear as `vda`/`vda1` (virtual disks) or `sda`/`sda1` on other systems, and can even be `cat`'d (produces binary output тАФ stop it with Ctrl+C).
- **`/etc`** holds system configuration files (and some applications store config there too); `/etc/network/interfaces` is a file holding network interfaces and their assigned IP addresses, historically edited directly to change network settings (still common on some systems).
- **`/media`** is where the system automatically mounts drives (e.g. a plugged-in USB flash drive); **`/mnt`** is where you manually mount drives using commands. Both mount drives, represented as files.
- The Linux file system's official name/arrangement is the **FHS тАФ Filesystem Hierarchy Standard** (Chuck: "file system hierarchy").
- Commands introduced this episode: `whoami`, `cat` (= "concatenate"), `cp` (copy), `rm` (remove), `sudo`, `adduser`, `which`, `clear`.
