# The AI Attack Blueprint (Interview with Jason Haddix)

## Summary

Interview in which Chuck Keith hosts AppSec/AI-security expert **Jason Haddix**
(guest — context, not persona training) for a roughly one-hour walkthrough of how
modern AI-enabled applications get attacked. Haddix lays out the methodology his team
uses for "AI pen testing" (distinct from narrower "AI red teaming"), a prompt-injection
taxonomy they released across conferences (RSA and adjacent events), and a defense-in-depth
stack for building AI apps more securely. Chuck plays the curious-generalist interviewer:
he repeatedly frames AI security as a new "wild west," admits he is "still very new" to
it, asks how attacks are discovered, and openly says he will "selfishly steal" the secure
stack for his own projects.

The bulk of the technical content is Haddix's expertise. Chuck's contribution is framing,
astonishment, and steering questions (MCP security, agent-to-agent, autonomous offensive
agents, secure stacks, local-model practice, favorite frontier models). Only Chuck's
questions/framing train the persona.

## Key claims (dated — Haddix-guest vs Chuck)

- **[2025-08-12 | Haddix-guest]** "AI pen testing" is broader than "AI red teaming":
  red teaming just tries to make the model say bad things; a pen test is a holistic
  security test of the whole AI-enabled app. (paraphrase)
- **[2025-08-12 | Haddix-guest]** Their methodology has ~six attack areas: identify
  system inputs, attack the ecosystem (DevSecOps/observability apps around the model),
  attack the model (red-teaming + business-integrity attacks like coercing discounts/returns),
  attack the prompt engineering (business logic now lives in prompts; leaking it is high value),
  attack the data (RAG/CAG stores often contain un-redacted PII/trade secrets), and attack
  the application (XSS, RCE, streaming misimplementation). Prompt injection is the vehicle
  throughout; the job resembles a mashup of web pentest + red team + AI-specific attacks. (paraphrase)
- **[2025-08-12 | Haddix-guest]** In Canada, an enterprise chatbot is legally treated as a
  representative of the company, so it must honor what it says — hard to enforce on the model.
  US case law is still up in the air. (paraphrase, legal claim — unverified here)
- **[2025-08-12 | Haddix-guest]** No guardrail/classifier they have tested is foolproof;
  they have at least partially bypassed all of them, including Nvidia's Nemo Guardrails. (paraphrase)
- **[2025-08-12 | Haddix-guest]** At an OpenAI conference, Sam Altman (asked by Daniel Miessler)
  said prompt injection might reach ~95% solved but is not solvable today without a leap beyond
  transformer/attention architectures; Haddix reads this as prompt injection being around "for a
  long, long time." (paraphrase, second-hand attribution → uncertain re: Altman's exact words)
- **[2025-08-12 | Haddix-guest]** Their prompt-injection taxonomy has four parts: **intents**
  (~21-22 goals, e.g. leak the prompt, poison data, discover API endpoints, test for bias,
  business integrity), **techniques** (narrative injection, token smuggling, end sequences,
  nesting payloads), **evasions** (meta-character confusion, reverse Unicode, obscure/fictional
  languages, emoji-hidden Unicode) that defeat classifiers, and **utilities**. (paraphrase)
- **[2025-08-12 | Haddix-guest]** Much of real AI pentesting is **manual**, not brute-force
  automation, because attacks must fit the client's business context; AI assists once the
  context is known. Tools mentioned: their in-progress (to-be-open-sourced) tool "Ronin,"
  plus existing ones like Garak and PyRIT. (paraphrase)
- **[2025-08-12 | Haddix-guest]** MCP made security worse, not better: MCP servers often ship
  with no security, no role-based access control, over-scoped read/write API access; Amazon +
  Intuit's paper "Enterprise-grade security for the Model Context Protocol" provides a threat
  model. Agent-to-agent is too new to have seen deployed but will need the same retrofitting. (paraphrase)
- **[2025-08-12 | Haddix-guest]** Exfiltration tricks: **link/image smuggling** (base64-encode
  stolen data into a markdown image URL pointing at a logging server), the **emoji Unicode-metadata**
  method, **"Bjection"** (teach the model a custom character-to-number encoding to bypass in/out
  classifiers, from Haze Labs), and a **"syntactic anti-classifier"** GPT that rephrases blocked
  image prompts via synonyms/metaphor. Code/link contexts get laxer classification because models
  are meant to help with code. (paraphrase)
- **[2025-08-12 | Haddix-guest]** Autonomous offensive-security agents are closer than he thought;
  he saw demos of agents scoring high on monthly bug-bounty leaderboards, but human specialists
  still win on creativity; agents will handle continuous low/mid-tier testing. (paraphrase)
- **[2025-08-12 | Haddix-guest]** Recommended secure AI-app stack (defense in depth, ~90-92%,
  not foolproof): strong web-layer input validation/output encoding (Node etc.), careful websocket
  streaming, an up-to-date frontier model, a well-tested classifier/guardrail on input AND output,
  least-privilege/role-scoped API keys (read-only where possible), and business-logic protection in
  the system prompt. Agentic systems multiply the cost/latency because each agent needs protecting. (paraphrase)
- **[2025-08-12 | Haddix-guest]** To learn prompting, reverse-engineer leaked system prompts of
  major products (Windsurf, Cline, Cursor, GPT-4o, Claude); repos named include ASG's "system prompt
  leaks." Learning resources: Bossy Group / Pliny the Prompter's "L1B3RT4S" GitHub, Mozilla's "Odin"
  bug-bounty (paying for model-level issues like the ASCII-encoded Amazon Rufus sarin-gas bypass),
  and CTFs like Lakera's "Gandalf." (paraphrase)
- **[2025-08-12 | Haddix-guest]** Favorite frontier models: Gemini if forced to pick one; OpenAI
  o1 Pro (deep research) is "the king" for research; also Manus (general agent), DeepSeek R1 671B
  (slept-on research model, usable free via Lambda), and his own multi-model tool "MultiDoc."
  Don't trust leaderboards/Chat Arena — vendors game them; test models yourself. (paraphrase)
- **[2025-08-12 | Chuck]** Chuck frames AI security as a lawless new frontier: existing security
  protocols don't cover AI, so it is "the wild west," and repeatedly says he cannot even imagine how
  to defend against the attacks Haddix describes. (paraphrase)
- **[2025-08-12 | Chuck]** Chuck positions himself as a fast-learning generalist: when talking to
  developers he is "the expert," yet tells Haddix "you should know more than me" — self-reported. (paraphrase)
- **[2025-08-12 | Chuck]** Chuck admits an N8N "rabbit hole" problem: he deployed N8N for the first
  time that week, had to stop himself, journaled about getting distracted by it — self-reported
  personal habit/workflow detail. (paraphrase)
