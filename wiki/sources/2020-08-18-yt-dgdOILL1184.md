# new Kali Linux GUI on Windows 10 (WSL 2) // 2020.3 Release

## Summary

Chuck walks through two features of the Kali Linux 2020.3 release. The headline feature is **Win-KeX** (the Kali Desktop Experience for Windows), which adds a full graphical desktop to Kali running under WSL 2 on Windows 10 — launched with the single command `kex`, so Kali behaves "like you're running an app on Windows." The second feature is the pending shift of Kali's default shell from Bash to **Zsh**: Zsh ships ready and installed in 2020.3, but the official default-shell migration doesn't land until 2020.4. He demonstrates several Zsh conveniences (automatic `cd`, command autocomplete via the right-arrow, and `sudo !!` to re-run a failed command with sudo) and shows the config/install steps for fresh installs, upgrades, and manual installs. The video assumes Kali is already installed on WSL 2 (covered in his prior video) and closes by noting the release also added Bluetooth support and other minor tweaks.

## Key claims (dated — setup)

All setup steps are as demonstrated for Kali Linux 2020.3 on 2020-08-18 (version-specific; may not apply to later releases).

- Kali Linux 2020.3 is a new release with two features Chuck highlights: an upgraded WSL 2 GUI (Win-KeX) and the move toward Zsh as the default shell. (paraphrase)
- With Kali installed under WSL 2, launching from Windows is done by typing "kali" in the Windows search bar; this drops into the Kali command line. (paraphrase)
- Typing the single command `kex` from the Kali command line launches the full graphical desktop experience. (paraphrase)
- "Kex" refers to **Win-KeX**, which stands for the Kali Desktop Experience for Windows. (paraphrase)
- Prerequisite: Kali Linux must already be installed on WSL 2 (covered in his previous video); a viewer can skip the GUI part of that prior video and come straight to Win-KeX. (paraphrase)
- To exit the desktop experience: press F8, then choose "exit viewer." (paraphrase)
- Win-KeX install is one command after updating: `sudo apt update && sudo apt upgrade`, then `sudo apt install kali-win-kex -y`. (paraphrase)
- After install, the GUI is launched by typing `kex`. (paraphrase)
- Zsh is ready and installed in 2020.3, but the official migration from Bash to Zsh as the default shell won't happen until 2020.4. (paraphrase)
- On 2020.3 the shell defaults to Bash; to switch to Zsh in the current session, type `zsh`. (paraphrase)
- Zsh advantage — automatic cd: you can change directory by typing the directory name without the `cd` prefix. (paraphrase)
- Zsh advantage — command autocomplete: as you type, Zsh suggests the command and the right-arrow key completes it. (paraphrase)
- Zsh advantage — `sudo !!`: after a command fails for lack of privileges, typing `sudo !!` re-runs the previous command with sudo. (paraphrase)
- If you upgraded to 2020.3 (rather than a fresh install), copy the default Zsh config with `cp /etc/skel/.zshrc ~/` to get the config Kali intends you to have. (paraphrase)
- To install Zsh manually along with recommended plugins: `sudo apt install zsh zsh-syntax-highlighting zsh-autosuggestions`. (paraphrase)
- Zsh supports many additional plugins and is described as a huge asset for Linux power users and for hacking with Kali. (paraphrase)
- Kali 2020.3 also added Bluetooth support and other minor tweaks and enhancements. (paraphrase)
