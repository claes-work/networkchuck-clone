# Access ANY Network (remotely)

## Summary

Chuck demonstrates how to get full remote access to a network by placing a small
device inside it (a Raspberry Pi 4, but "really anything" — Pi 5/3/Zero, a NAS, a
laptop) running a **Twingate connector**. The framing device: he is the "unofficial IT
guy" for his church (named "Reach," in Rockwall), and he wants persistent remote access
to support it — plus the same setup doubles as a pen-testing drop box. The pitch is
walk in, plug in via ethernet or Wi-Fi, walk away, and you're in from anywhere.

He stresses repeatedly that **this is NOT a VPN** — it's Twingate, a zero trust network
access (ZTNA) solution, "so much better." No firewall config, no CGNAT, no port
forwarding, no inbound rules to open. Setup flow: create a Twingate network (lives in
the cloud, Twingate manages it), pick an identity provider (he uses Google), define a
**remote network** ("the church"), and deploy a **connector** onto the Pi. The Pi is
flashed with Raspberry Pi OS 64-bit via Raspberry Pi Imager (pre-baking hostname,
user, and Wi-Fi), then the connector is installed with a single copy-paste tokenized
Linux command.

The "something new" beyond a normal Twingate tutorial: a **Python script using
Twingate's API** that auto-discovers the network the connector landed in and
auto-creates resources from the connector's private IP — useful when you drop the
device in a network whose IP scheme you don't know (he notes you can also just manually
add the resource if you already know the subnet). He then uses the Twingate client
(available Windows/Mac/Linux/iPhone/Android) to SSH into the Pi, runs an **Nmap scan**
of the /24, finds hosts (a Windows box, an iMac, a printer, the router), and adds
discovered hosts (e.g. the printer on port 80/443) as tightly-scoped resources with
DNS aliases (`printer.reach.rockwall.local`).

He closes on Twingate's security/governance features — least privilege (resources
default to no access until explicitly granted), device posture policies (only trusted
devices, only Mac, disk encryption / firewall required, restrict by serial number),
access expiration and auto-lock, activity logs and reports, and **service accounts**
(headless clients — e.g. running Uptime Kuma to monitor the printer without opening any
firewall rules). Suggested use case: drop cheap Pis at family members' houses for
permanent remote support ("Merry Christmas. I control your network to help you").

Sponsored by Twingate (a long-running channel sponsor he says he uses daily in his own
business); free for up to five users.
