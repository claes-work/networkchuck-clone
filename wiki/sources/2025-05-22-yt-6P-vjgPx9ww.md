# 37 INSANE Linux Commands you NEED to Know in 2025

## Summary

A rapid-fire roundup of 37 command-line tools Chuck recommends for 2025, weighted heavily toward modern (often Rust-powered) replacements for classic Unix utilities and toward terminal UIs (TUIs) that make CLI work visual and interactive. Recurring themes: swap old tools for prettier/faster modern equivalents (`ripgrep` for grep, `fd` for find, `eza` for ls, `zoxide` for cd, `dog` for dig, `procs` for ps, `duf` for df), lean on TUIs for Docker/packet capture/file management (`lazydocker`, `termshark`, `ranger`), and finish with AI-in-the-terminal (`fabric`, local `ollama` agents) to generate commands on the fly. Demonstrated on a Hostinger VPS (the sponsor). Chuck's most-hyped picks: `lazydocker` ("probably my new favorite tool") and `asciinema` ("hands down the most amazing command on this list").

## Key claims (2025-05-22, paraphrase unless quoted)

- **Disk/space:** `ncdu` — visual, interactive disk-usage explorer; "like df and du but better," can delete and drill into folders. `duf` — a prettier `df` for drive usage.
- **Search:** `ripgrep` (rg) — Rust-powered grep replacement, very fast for finding log errors, functions, or files mentioning an API key. `fd` — a better `find`: recursive, case-insensitive, ignores hidden files, colorized, often faster. `fzf` (fuzzy finder) — interactive filtering of any list or pipe; powerful chained (e.g., pipe history into it to kill a process).
- **Remote/connectivity:** `mosh` — SSH-like but supports roaming (Wi-Fi to cellular) and keeps the session open; must be installed on both sides. `rsync` — smart, delta-only file transfer that resumes broken transfers and mirrors over SSH; "the best way to copy directories or large files." `wormhole` (magic-wormhole) — peer-to-peer, end-to-end encrypted file transfer via a code, optionally relayed through a firewall.
- **Hardware/system info:** `lshw` — full hardware inventory (filterable by class, e.g. CPU/memory/network/disk). `stat` — every detail about a file including creation time; `-f` shows filesystem info. `systemd-analyze blame` / `critical-chain` — diagnose slow boot by service time and dependency bottlenecks.
- **Monitoring/stats:** `glances` — all-in-one stat dashboard with a web/API mode. `iotop` — top-style view of processes causing disk I/O. `dstat` — combined timeline of CPU/RAM/disk/network (combines vmstat, iostat, netstat). `watch` — reruns any command on an interval (e.g., `watch -n 0.5 nvidia-smi` to monitor a GPU). `progress` — shows progress of running commands like a secure copy.
- **Networking/DNS:** `mtr` — "ping and traceroute have a baby"; live hop-by-hop latency and packet loss. `dog` — a colorized `dig` replacement supporting DNS-over-TLS and JSON output. `termshark` — terminal UI for tshark (mouse-driven, filter by DNS, save/reload pcap). `lsof -i` — show the process owning a given port (e.g., port 22, port 80). `ipcalc` — quick subnet calculator (ranges, mask, wildcard from CIDR).
- **Navigation/files:** `zoxide` (z) — smarter `cd` that ranks frequent/recent directories; `zi` uses fzf for interactive selection. `eza` — better `ls` with color coding, built-in tree view, and icons. `ranger` — TUI file manager with Vim keybinds, bulk renames, and previews.
- **Process/Docker:** `procs` — prettier `ps` (sort by CPU, tree view). `lazydocker` — one-command interactive TUI for Docker; "probably my new favorite tool."
- **Data deletion:** `shred` — overwrites a file over multiple passes before deleting to thwart recovery (noted that `rm` only unlinks, leaving data until overwritten).
- **Utilities:** `moreutils` — bundle including `ts` (add timestamps), `errno` (look up error numbers), `ifdata` (cleaner interface info), `vidir` (edit directory names in your editor), `vipe` (insert your editor mid-pipeline). `unp` — unpacks any archive by guessing the right command from the archive type. `jq` — "the JSON lightsaber" for querying/transforming JSON. `task` (Taskwarrior) — task management from the CLI.
- **Recording:** `asciinema` (asciinema / agg) — record terminal sessions to small text-based cast files, replay them (pausable, copyable text), self-host an upload server, and convert recordings to GIF via `agg`. Called "hands down the most amazing command on this list."
- **AI in the terminal:** `fabric` — CLI to interact with AI, combined with known commands (e.g., pipe history to summarize most-used commands, analyze open ports or syslog). Local `ollama` — build a custom agent whose only job is crafting commands on the fly.
