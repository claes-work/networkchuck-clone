# Kali Linux APPS on Windows in 5min (WSLg)

## Summary

A ~5-minute tutorial in which Chuck Keith shows how to run Linux — including full
graphical Linux apps — natively on Windows using WSL 2 (Windows Subsystem for Linux)
and WSLg (its GUI layer). He frames WSL 2 as "Windows and Linux getting married": the
easiest way to run Linux on Windows, and WSLg as the newer, even-crazier feature that
lets individual Linux apps appear as part of the Windows GUI. The video walks through
prerequisites, a one-command install, installing Ubuntu (the default distro) and Kali
Linux, and getting GUI apps running on both Windows 11 (via WSLg) and Windows 10 (via
a VNC/win-kex workaround). Sponsored by Private Internet Access (VPN).

The core pitch: WSL is the coolest way to get people invested in Linux — install it
like an app with one command and start playing.

## Key claims (dated — setup, 2022-03-04)

- Prerequisites: a Windows computer running Windows 10 or Windows 11 — nothing else
  needed. Windows XP is not supported.
- Windows 10 requires version 2004 or higher, or build 19041 or higher. Windows 11
  needs no specific version beyond being Windows 11.
- WSLg (GUI app support) is only available on Windows 11. Windows 10 still runs WSL 2
  fine but not WSLg; GUI on Windows 10 is achieved later via win-kex.
- To check your Windows 10 version, type `winver` in the search bar and hit enter.
- Virtualization must be enabled in the BIOS. Enter the BIOS (commonly by rebooting
  and rapidly pressing the Delete key). On AMD systems look for a CPU setting called
  SVM; on Intel systems look for Intel Virtualization / Intel VT. Enable it, then save
  and reboot. Exact steps vary by motherboard, BIOS version, and CPU.
- Installing WSL 2 is a single command, identical on Windows 10 and 11: launch an
  elevated (run-as-administrator) command prompt and run `wsl --install`, then reboot.
  On Windows 11 this also downloads and installs GUI app support (WSLg); on Windows 10
  that step is absent.
- By default the install continues by installing Ubuntu; you then set a username and
  password to finish. Ubuntu then runs like an app on Windows.
- To demo WSLg on Windows 11: in the Ubuntu terminal run `sudo apt update`, then
  install a GUI app, e.g. `sudo apt install gimp -y` (GIMP is described as "Photoshop
  for Linux"). The installed Linux app then appears in the Windows Start menu and can
  be launched, moved, and minimized like a native Windows app. Chuck also installs
  `gedit` as a Linux text editor.
- To install other distros: run `wsl --list` to see installed distros (Ubuntu is the
  default), then `wsl --list --online` to list installable distros. This command does
  not need an elevated prompt.
- Distros are installed with `wsl --install -d <distribution>`. Chuck warns NOT to
  install Kali this way (`wsl --install -d kali-linux`), because the version WSL pulls
  is not the current Kali and will cause major issues.
- Recommended Kali workaround: install Kali Linux from the Microsoft Store (search
  "windows store", then search "Kali Linux") to get the latest working version. Works
  on both Windows 10 and 11. Set a username and password on first launch.
- To get GUI working in Kali, install win-kex on both Windows 10 and 11:
  `sudo apt install kali-win-kex -y`. During install it may prompt for language
  (choose English). On Windows 11 this enables the WSLg GUI; on Windows 10 it provides
  the GUI via VNC.
- On Windows 11 after win-kex, the Windows Start menu shows a Kali Linux folder full
  of Kali apps (e.g. a text editor, OWASP ZAP) that run in Windows.
- On Windows 10, launch the full Kali desktop with `kex --win -s`. It prompts to set a
  VNC password (you can say no to setting one). This displays the entire Kali Linux
  desktop on Windows; press F8 and choose disconnect to exit.
- WSLg can also run Linux apps that are games, with working sound — Chuck installs and
  runs Hedgewars (in Ubuntu) as a demo.
