# The Dark Web EXPOSED (FREE + Open-Source Tool)

## Summary

Chuck demonstrates **Robin**, a free, open-source AI tool for searching, scraping, and
researching the dark web, built by a dark-web threat researcher named **Apurv** whom
Chuck met at DEF CON in Las Vegas. The video is framed as a threat-researcher / OSINT
tutorial: Chuck explains why the dark web is hard to research (Tor's slow, breakable
onion-relay circuits; criminal sites that are only online a couple of unpredictable days
a week), then walks through installing and running Robin.

Robin's pitch is that it compresses a six-to-eight-hour manual research marathon into
roughly a 30-minute session. It works in stages: it refines the user's search query with
AI, scours the dark web across multiple search engines (returning 900+ raw results), uses
AI to filter those down to ~20 verifiable real sources, scrapes those sites, and then uses
AI to summarize findings and suggest next research steps. Results can be exported as a
markdown file (e.g., into an Obsidian vault).

Setup requires Tor (`apt install tor` / `brew install tor`) and Docker; the Robin repo is
cloned from GitHub and built locally into a Docker container (not published on Docker Hub).
Robin is model-agnostic — it supports Anthropic, OpenAI, Google, and local models like
Llama 3.1 via a `.env` file of API keys. It is command-line-first but ships a web GUI
(localhost:8501). Chuck runs a live demo searching "ransomware" and "Conti," surfacing
darknet forum listings, threat-actor info, and cryptocurrency addresses (DOGE, LTC, BTC,
XMR).

A large portion of the video is devoted to **safety and ethics**: use a VPN plus Tor, never
access illegal marketplaces or search for CSAM/drugs/violence-for-hire content (Chuck stresses
severe legal consequences, especially in the US), and don't download anything from the dark
web. Apurv built guardrails into Robin's prompt to refuse not-safe-for-work / illegal queries,
though Chuck notes it isn't foolproof. Chuck also relays Apurv's account of real threat-research
tradecraft: sock-puppet accounts, burner phone numbers, maintaining a consistent undercover
persona, and the long waiting game of building trust in forums over days or weeks. Chuck
justifies showing the tool on two grounds — education (understanding threats to defend against
them) and letting viewers check whether their own data (email, name) is exposed on the dark web.
The sponsor segment (ThreatLocker) covers zero-trust endpoint protection and "ring fencing"
(e.g., blocking Word from calling PowerShell). The video ends with Chuck's signature prayer.
