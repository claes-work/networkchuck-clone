# UNLIMITED CLOUD STORAGE - Synology + Crashplan - $10/month

> ⚠️ **DATE-SENSITIVE (2019).** Pricing and service details below reflect May 2019.
> CrashPlan has since changed its consumer/small-business offerings and branding
> (the "CrashPlan for Small Business" / ProE $10/month unlimited plan described here
> should be treated as historical, not current pricing/availability).

## Summary

A homelab/storage tutorial in which Chuck sets up "unlimited" cloud backup by pairing
a Synology NAS with CrashPlan's small-business plan, using a third-party Docker
container to bridge the two (Synology has no native CrashPlan app). Chuck frames the
video around the "3-2-1" backup philosophy — he cites the maxim that if your data
doesn't exist in three places, it doesn't exist — and the danger of a single point of
failure (his old single external hard drive, or even the NAS itself if the whole box
dies).

Hardware: a Synology DS219+ (Chuck also verbally calls it a "cloud station 1019 plus"
and mentions "five bays," but the on-screen unit he names is the ds2 19 plus) with
three 6 TB drives configured as one volume. The backup target is CrashPlan's small-
business plan at $10/month per device — and because the NAS counts as one device,
Chuck can dump all his data to it and back up the whole thing for that single price.

The walkthrough: install Docker from Synology's Package Center → download a community
CrashPlan PRO (small business) Docker container from the Docker registry → launch it
with high privilege and a memory limit, mapping a Synology shared folder to the
container's `/storage` path → access CrashPlan's web UI at the NAS hostname on a
Synology-assigned port (not CrashPlan's native 5800) → create a dedicated Synology user
for CrashPlan with read/write access → SSH in (via PuTTY) to get that user's UID/GID
(UID 1028, GID 100 in his case) → enter those into the container's environment
variables so the container has permission to read the files → select folders and start
backing up. Chuck notes the container sometimes fails to log in on first start and the
port can change after a restart. He positions CrashPlan as best for personal/small-
business users and recommends Azure/AWS/Google Cloud for medium-to-large businesses
(teasing a future Azure-backup video). Includes a signature coffee detour (Noble Coyote
Coffee Roasters, Deep Ellum, Dallas — unsponsored) and an embedded clip from his Azure
training course explaining containers vs. VMs.

## Key claims (dated — 2019-era setup, pricing, and service)

- **Backup philosophy:** Data that doesn't exist in three places doesn't exist; a
  single hard drive (or a single NAS box) is a single point of failure. Cloud backup
  gets your data out of your house. [paraphrase]
- **Service/pricing (2019, historical):** CrashPlan's small-business plan ("PRO Online"
  / ProE) was $10/month **per device** for unlimited storage. Because the NAS registers
  as one device, backing up the whole NAS costs that single $10/month. [paraphrase — pricing likely obsolete now]
- **Why Docker is needed:** Synology's native cloud-backup integrations (Azure, AWS,
  Google Cloud, etc.) do not include CrashPlan, so a third-party CrashPlan-PRO Docker
  container is used to bridge the gap. Anything that supports Docker can run this
  container. [paraphrase]
- **Hardware:** Synology DS219+ (named on screen), 8 GB DDR3 RAM, quad-core CPU, three
  6 TB drives as one volume. Beyond storage it also serves as his Plex server and a
  backup target for VMs. [paraphrase — note verbal model-name inconsistencies in transcript]
- **Setup steps:** Install Docker via Package Center → pull the CrashPlan-PRO container
  from the Docker registry (choose latest) → launch with high privilege + 1 GB memory
  limit → map a Synology folder to the container's `/storage` path → reach the web UI at
  `<nas-host>:<synology-assigned-port>` (Synology remaps CrashPlan's native 5800). [paraphrase]
- **Permissions gotcha:** The container can't see files until permissions are set.
  Create a dedicated Synology user for CrashPlan with read/write on the target folder,
  SSH in (PuTTY) and run `id <user>` to get the UID (1028) and GID (100), then enter
  those into the container's environment variables and restart. [paraphrase]
- **Known quirks:** The CrashPlan container sometimes doesn't log in on first startup —
  restart the container and log in on the second try; the mapped port may change after a
  restart. [paraphrase]
- **Fit guidance:** CrashPlan suits personal/small-business users; medium-to-large
  businesses should prefer Azure/AWS/Google Cloud as more robust (Azure-backup tutorial
  teased for a later video). [paraphrase]
- **Disclosure:** Synology provided the NAS unit for free; the video is not sponsored by
  Synology or CrashPlan (Chuck states both). [paraphrase]
