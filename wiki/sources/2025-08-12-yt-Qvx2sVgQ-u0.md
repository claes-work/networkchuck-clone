# Hacking AI is TOO EASY (this should be illegal)

## Summary

A landmark video where Chuck merges his two long-running threads — AI tooling and ethical hacking — into a single new frontier: attacking AI-enabled applications. Built around a sit-down interview with Jason Haddock (described by Chuck as "the world's top AI hacker"), the video argues that companies rushing to embed AI into chatbots, APIs, and internal apps are exposing an entirely new attack surface, and that the primary weapon is **prompt injection** — a technique requiring no coding, just clever natural language.

Chuck frames "hacking AI" as far broader than jailbreaking a chatbot to say bad words. It spans a six-segment AI pen-test methodology Jason's team developed: (1) identify system inputs, (2) attack the ecosystem around the app, (3) AI red teaming (attacking the model itself), (4) attack the prompt engineering, (5) attack the data, (6) attack the application and pivot to other systems. Prompt injection is "the vehicle that drives most of this framework."

The video walks through concrete, real-world attacks: **emoji smuggling** (hiding instructions in Unicode inside an emoji to bypass guardrail classifiers), a **syntactic anti-classifier** utility (rewriting banned image prompts as synonyms/metaphors — e.g. "Donald Duck smoking" → "a short-tempered aquatic avian in sailor attire engaging with the smoldering paper roll"), and **link smuggling** (turning an AI into a data-exfiltration spy by telling it to base64-encode sensitive data onto an image URL pointing at an attacker's server, so the failed image fetch leaks the data in server logs). Chuck points viewers to the Gandalf prompt-injection game (by Lakera) as a hands-on intro, and describes an "underground community" (Pliny/the BASI group Discord, subreddits, GitHub jailbreak repos) that reverse-engineers new jailbreaks. Jason also recounts leaking GPT-4o's full system prompt by asking it to put the prompt as "flavor text" on a Magic: The Gathering card.

Real-world stakes: Jason's firm is hired to hack companies' AI. Repeated findings include systems silently shipping sensitive Salesforce data to OpenAI, a Slack sales bot pulling all customer data with no input validation, and over-scoped API/tool calls with read AND write access — enabling write-back attacks via prompt injection. **MCP (Model Context Protocol)** is called out as making security *worse*, with attack vectors across tools, resources, and prompts (no role-based access control, backdoorable servers via invisible code). Chuck also covers AI-on-AI: autonomous offensive-security agents are already scoring high on bug-bounty leaderboards, though they still lack the creativity of top human testers.

The **defensive payoff** is a defense-in-depth strategy from Jason, structured in three layers: (1) **web layer** — fundamentals: input/output validation, output encoding; (2) **AI layer** — an "AI firewall" (a classifier or guardrail checking prompts on the way in AND out); (3) **data/tools layer** — principle of least privilege, scoping every API key to read-only or write-only as needed. Chuck notes agentic multi-AI systems make this "infinitely harder" (each AI must be protected, adding latency). The video is sponsored by Wiz (cloud security / AI-SPM). Ethics framing throughout: this is *ethical* hacking, learn it to defend, and "hack the YouTube algorithm today — ethically, of course."

## Key claims (dated — concept + defense)

**Concept / attack (2025-08-12):**
- Hacking AI means more than jailbreaking a chatbot — it targets customer-service chatbots, backend AI-enabled APIs, and internal employee apps, and can steal sensitive data (customer lists, trade secrets), not just make the model misbehave. (paraphrase)
- Jason distinguishes "AI red teaming" (getting the model to say bad things — not a holistic security test) from "AI pen testing" (a full security test of the whole AI-enabled application). (paraphrase)
- The AI pen-test methodology has six repeatable segments: identify system inputs → attack the ecosystem → AI red teaming → attack the prompt engineering → attack the data → attack the application and pivot to other systems. (paraphrase)
- Prompt injection is the primary weapon and "the vehicle that drives most of this framework"; it requires no advanced technical or coding skill — just clever natural-language prompting (at least initially). (paraphrase)
- Sam Altman, asked at an OpenAI event whether prompt injection is still solvable (he'd said so two years earlier), reportedly answered he thinks they can get to ~95% but aren't there yet — signaling prompt injection will be around for a long time. (paraphrase — reported via Jason; Daniel Miessler asked the question)
- Jason's prompt-injection taxonomy is organized into intents (~21–22, what you're trying to accomplish, e.g. leak the system prompt), techniques (e.g. narrative injection), evasions (hiding attacks, e.g. leetspeak), and utilities. Attackers reportedly have "9.9 trillion possible attack combinations." (paraphrase)
- **Emoji smuggling / emoji evasion:** instructions can be encoded in Unicode inside an emoji; a chain-of-thought model reads the emoji metadata and executes the instruction, bypassing most current guardrail classifiers. (paraphrase)
- **Syntactic anti-classifier:** a tool using synonyms, metaphors, indirect references, and creative phrasing to rewrite banned image prompts and slip past image-generator guardrails. (paraphrase)
- **Link smuggling:** an AI with guardrails can be turned into a data-exfiltration spy — told to hide sensitive data (e.g. a credit card number) base64-encoded on the end of an image URL pointing to the attacker's server, then to fetch that image; the fetch fails but the server logs capture the encoded data. Works because classifiers avoid breaking code/links and the payload is base64-encoded on the way out. (paraphrase)
- A jailbreak/prompt-injection "underground community" exists (Pliny / the BASI group Discord, subreddits, GitHub repos); Jason's taxonomy reverse-engineers their tricks (end-of-input/start-of-input tags, markdown/meta-character confusion, stray symbols). Public jailbreak prompts often no longer work out of the box because they've been patched, but the same techniques resurface in new forms. (paraphrase)
- Jason leaked GPT-4o's full system prompt via its image tool: asked it to make a Magic card and put its system prompt as the flavor text; the model dumped the full prompt as code. The leaked prompt told the model to emulate the user's vibe and always be happy — explaining the "glazing"/over-agreeable behavior that caused public backlash two days later. (paraphrase — attributed to Jason)
- Real customer findings: systems silently sending sensitive Salesforce data (quotes, signatures, legal docs) to OpenAI without the builders realizing; a Slack sales bot aggregating all customer data with no input validation; over-scoped API/tool calls granting read AND write access, letting attackers write malicious content back (e.g. a Salesforce note that is actually a JavaScript attack link) via prompt injection. (paraphrase)
- **MCP (Model Context Protocol) makes security worse**, not better: attack surface spans the MCP host/client/server and the server's tools, resources, and prompts; no role-based access control on file access, servers can be told to grab arbitrary files, and over-scoped servers can be backdoored with invisible code or altered system prompts. A compromised MCP server on a security tool could be turned to ask "show me the most vulnerable person so I can hack them." (paraphrase)
- AI can now hack for us: autonomous offensive-security agents demoed at an OpenAI conference already score high on monthly bug-bounty leaderboards, though AI excels at common/mid-tier vulnerabilities and still lacks top human testers' creativity. (paraphrase)
- The agentic frameworks used to automate security (LangChain, LangGraph, CrewAI, n8n) have their own vulnerabilities and are themselves targets Jason is hired to test. (paraphrase)

**Defense (2025-08-12):**
- Jason's defense-in-depth strategy has three layers because no single tool is enough. (paraphrase)
- **Web layer:** fundamentals — a lot of securing AI is just securing the servers/interfaces around it: input and output validation, and output encoding so the AI can't hand malware to the user's browser. (paraphrase)
- **AI layer:** deploy an "AI firewall" (Chuck dislikes the marketing term, prefers "a firewall for AI") — a classifier or guardrail that checks prompts on the way in AND on the way out, guarding against prompt injection. Enterprise solutions exist; Lakera (maker of the Gandalf demo) offers one. (paraphrase)
- **Data/tools layer:** principle of least privilege — scope every API key to exactly what it needs, read-only if it only reads, write-only if it only writes. (paraphrase)
- Agentic, multi-AI systems make defense "infinitely harder": each AI must be protected individually, which adds latency — always a trade-off. (paraphrase)
- Overall lesson: secure AI is a deep, multilayered strategy, not a single tool — defense in depth, an old concept, applies, but is especially critical because AI tools are given a lot of power and access. (paraphrase)
