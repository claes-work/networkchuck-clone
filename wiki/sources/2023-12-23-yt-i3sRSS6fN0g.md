# the most secure OS in the world.....I hate it

## Summary

Chuck walks through **Qubes OS** (the auto-captions consistently render it as "Cube's OSS"), which he calls "the most secure operating system in the world" — while noting that Qubes itself modestly self-describes as only a "reasonably secure operating system." He explains Qubes' core security philosophy: it assumes you *will* get hacked and that all software inherently contains exploitable bugs, so instead of trying to prevent compromise it works to **confine, control, and contain the damage** through isolation.

The bulk of the video is a tour of the Qubes model — running everything inside isolated, color-coded virtual machines called "qubes" — followed by a hands-on install guide for both bare metal and (unofficially) a VM. Chuck's honest verdict is admiration mixed with frustration: the isolation is genuinely powerful and Snowden-tier secure, but it's painful to set up, slow (it's running multiple VMs), and for his own use case he'd rather just spin up his own VMs manually. He ends by saying he'll stick with Windows, Linux, and Mac.

## Key claims

- Qubes OS is presented as effectively the most secure desktop OS, though Qubes itself only claims to be a "reasonably secure operating system" — Chuck reads the modest self-labeling as itself a sign of how paranoid/serious it is. (2023-12-23)
- Qubes' security model assumes compromise is inevitable and that all software contains exploitable flaws; rather than prevent hacks it isolates and contains the blast radius — "confine, control, and contain the damage." (2023-12-23)
- Qubes is essentially a **type 1 (bare-metal) hypervisor OS**, built on **Xen-based virtualization** (the open-source Xen Project hypervisor). (2023-12-23)
- Everything runs inside isolated compartments called **qubes**, which are essentially virtual machines. The user first picks an identity/context (e.g. "work you" vs. "personal you") and launches apps into that qube; each qube is isolated so a compromise in one cannot touch the others. (2023-12-23)
- **App qubes** are the VMs you launch apps in. Each is based on a **template qube** (a base OS plus commonly shared apps); updating an app in the template propagates the updated app to every qube using it. App qubes cannot modify their template, so a compromised app qube can't infect the template or other qubes — you just shut it down. (2023-12-23)
- Qubes are **color-coded** by trust level (e.g. work = blue, personal = yellow, weird = red) so you always know which qube you're interacting with. (2023-12-23)
- Templates shown include **Fedora**, **Debian 11**, and **Whonix** (rendered "Honix"/"Hun Xix" in captions); Whonix routes traffic through the **Tor** network for anonymous browsing. (2023-12-23)
- Even hardware and networking are isolated via **service/system qubes**: e.g. `sys-usb` for USB devices, `sys-net` for networking, `sys-firewall` for firewalling, and `sys-whonix` for Tor — connected to app qubes like Lego blocks. (2023-12-23)
- **dom0** is the admin qube — the most trusted, root-like master that manages all other qubes and the desktop/window manager. It has no internet connection and should not be used for anything else; if dom0 is compromised, the whole system is lost. (2023-12-23)
- **Disposable qubes** exist: anything created or browsed in them disappears when the qube is stopped. (2023-12-23)
- Hardware requirements are steep: a modern 64-bit CPU with virtualization (Intel VT-x + EPT, or AMD-V + RVI) and IOMMU (Intel VT-d) enabled; minimum 6 GB RAM (16 GB recommended); minimum 32 GB storage (128 GB recommended). Running Qubes *inside* a VM needs those resources on top of the host. (2023-12-23)
- Qubes officially does **not** support running inside a VM and only supports specific approved bare-metal hardware; Chuck installed it on an ASUS ROG laptop onto a portable SSD to avoid overwriting his existing OS. (2023-12-23)
- Install notes: write the ISO with **Rufus** in DD image mode (won't work otherwise); for a VM, **VMware Workstation Player** worked (VirtualBox failed, likely no IOMMU support) via nested virtualization; in BIOS, enable CPU virtualization and **disable Secure Boot** (re-enable it later if returning to your normal OS). Install takes ~30–45 minutes. (2023-12-23)
- Notable Qubes users cited: Edward Snowden. (2023-12-23)
- Chuck's honest security-vs-usability take: Qubes is cool, powerful, and genuinely the most secure, but it was a pain to set up and not the fastest (it's running several VMs). He argues that simply running your own VMs for specific tasks (e.g. a dedicated banking VM) might be easier and possibly more secure for him — and concludes he'll keep using Windows, Linux, and Mac, just "in a different way." (2023-12-23)
