# the $0.30 Hacking Lab

## Summary

Hands-on tutorial in which Chuck builds a fully-cloud hacking lab on AWS for roughly
30 cents per day, framed around a "cheap = accessible, no excuses" ethos. The lab has
two ingredients: free vulnerable target machines from VulnHub, and cheap on-demand
compute from AWS. Chuck argues you don't need a beefy gaming PC or a big home lab —
any device with a web browser (even an old laptop or iPad) is enough, because the
heavy lifting runs on Amazon's data centers and you pay only for the seconds you use.

The build: create an AWS VPC/network whose subnet matches the vulnerable box's
hard-coded IP range; launch a Kali Linux attack instance (T2 medium, ~4¢/hr) into the
public subnet with a key pair; download a VulnHub target (the "breach" box, a `.ova`
file); upload the `.ova` to an S3 bucket; use the AWS CloudShell command line to
create the IAM permissions (trust policy, role policy) and run `import-image` to
convert the `.ova` into an AWS AMI; deploy that AMI as a target instance in the
private subnet with all security-group restrictions removed; then SSH/GUI into Kali
and confirm the attack path with `ping`, `nmap`, and Firefox. When finished, stop or
terminate the instances so billing stops. Sponsored by ITProTV; Chuck credits Daniel
Lowry's "hands-on hacking" course as the inspiration.

## Key claims (dated 2022-06-03)

**Accessibility ethos**

- The only way to become a hacker is to start hacking immediately and start breaking
  things. (paraphrase)
- You need almost no hardware — the crappiest laptop, an iPad, or anything with a web
  browser works, because the compute lives in the cloud, not on your machine.
  (paraphrase)
- The one real "catch" is knowledge, not money — it's not easy, but the video walks
  you through every step. (paraphrase)
- His mission was to find the best lab at the lowest cost that is accessible to most
  people, if not everyone. (paraphrase)
- At ~4¢/hour, most people can afford this; building your own lab (rather than using
  pre-built ones) teaches special, valuable skills and makes you better. (paraphrase)
- You don't need a massive server, a gaming machine, or a paid lab service — you can
  set up a really cheap cloud lab with freely available resources and start hacking
  right away. (paraphrase)

**The build / cost model**

- VulnHub is a free, massive library of deliberately vulnerable machines ranging from
  easy to hard; you download the box and host it yourself. (paraphrase)
- The cloud gives an effectively unlimited lab — spin up many VMs recklessly because
  it's Amazon's hardware, not yours. (paraphrase)
- AWS bills per second of actual use: the running lab is ~4¢/hour, an 8-hour hacking
  marathon is ~30¢/day, and when stopped it costs almost nothing. (paraphrase)
- Instance sizing drives cost: T2 medium (2 vCPU / 4 GB) ~4¢/hr, T2 small ~2¢/hr; the
  target box can run on a tiny T2 micro. (paraphrase)
- AWS requires a credit card, but first-time users get a 12-month free tier that may
  make the lab free; you're only charged for what you use. (paraphrase)
- The VPC subnet must match the vulnerable machine's hard-coded network (he uses
  192.168.110.0/24) so the Kali attack box can reach the target. (paraphrase)
- The `.ova` file is the vulnerable VM; AWS converts it to its own AMI format via the
  CloudShell `import-image` task, which passes through statuses (pending → converting
  → completed). (paraphrase)
- Some VulnHub OVAs simply won't import/convert, but most work fine. (paraphrase)
- For the target instance, remove AWS's default protections — change the security
  group rule from SSH-only to "all traffic" from anywhere — because you're testing the
  box's vulnerabilities, not AWS security. (paraphrase)
- Stopped instances stop EC2 charges, but their disks still incur tiny EBS storage
  cost (he estimates roughly 3¢/month). (paraphrase)
- The lab is portable: with a VNC/remote app on a phone or iPad you can reach the same
  box from anywhere in the world; a Kali box in AWS can even download and upload the
  VulnHub OVA itself to dodge slow home internet. (paraphrase)
