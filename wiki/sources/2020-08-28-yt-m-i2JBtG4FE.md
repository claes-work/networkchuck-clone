# setup a FREE VPN server in the cloud (AWS)

## Summary

A step-by-step tutorial in which Chuck builds a personal, self-hosted VPN server on
AWS's free tier so viewers can protect their internet traffic without paying a
commercial VPN subscription. The build uses an EC2 virtual machine (a `t2.micro`
free-tier instance) launched from an AWS Marketplace AMI — an Ubuntu server with
**OpenVPN Access Server** pre-installed, chosen because installing OpenVPN by hand is
"kind of a pain." Chuck selects the free-tier-eligible, bring-your-own-license
Marketplace image (which grants two free client connections), generates and downloads
an SSH key pair, connects over SSH/PowerShell, runs initial setup by accepting the
defaults, and sets the admin/client password for the `openvpn` user with
`sudo passwd openvpn`.

He then logs into the OpenVPN web admin portal (`https://<public-ip>:943/admin`),
flips the routing setting so **all** client internet traffic is tunneled through the
VPN, and saves/updates the running server. On the client side he visits the user
portal (`:943`), downloads the pre-configured OpenVPN client for his device (Windows,
with iOS/Android options shown), connects, and verifies success with a "what's my IP"
search that shows his traffic exiting through the server (located in France in his
example) rather than his home IP.

The video closes by framing the build conceptually: connecting to this server gives
you secure access to your AWS **Virtual Private Cloud (VPC)** — reaching devices by
their private IP addresses instead of exposing public IPs — while also encrypting all
your internet traffic and letting you appear wherever your server is hosted. Chuck
stresses it is genuinely free for 12 months under the AWS free tier (750 hours/month
covers one always-on server, since a 31-day month is 744 hours), after which the
server costs roughly $10/month. He notes the same thing can be done on other cloud
providers but he's partial to AWS, and pitches his AWS Solutions Architect Associate
course (collaboration with Anthony Sacco and David Bombal) as the next step toward a
cloud engineering career.

## Key claims (dated — setup + concept)

All dated 2020-08-28 (source publication).

- You need a VPN to stay safe on the internet from threats like hackers — but you don't
  have to pay for one; you can host your own in the AWS cloud for free. (paraphrase)
- Prerequisites are just two things: a cup of coffee and a free AWS account. (paraphrase)
- **EC2** is Amazon's service for deploying a virtual machine / virtual server in the
  cloud, which is what will run the VPN server. (paraphrase)
- Rather than a bare-OS AMI, select the image from the **AWS Marketplace** to get
  pre-configured tools like OpenVPN already installed. Search "openvpn" and pick the
  free-tier-eligible Ubuntu + OpenVPN image (not the 10-connected-devices variant).
  (paraphrase)
- OpenVPN itself is free and open source, but **OpenVPN Access Server** is a paid
  commercial product; the bring-your-own-license version used here still gives two
  client connections for free — enough for you and a friend. Beyond that you'd need to
  pay. (paraphrase)
- Choose a **general purpose t2.micro** instance, which carries the free-tier-eligible
  icon, then review and launch. (paraphrase)
- Create and **download an SSH key pair** at launch (he names his "vpn server") — it's
  your only chance to download the key, and you need it to connect and configure the
  server. (paraphrase)
- Connect via the standalone SSH option; the connect command references the `.pem` key
  file, so adjust the path (e.g. prefix `downloads/vpnserver.pem`) so it points at
  wherever the key was downloaded. (paraphrase)
- Initial server setup is mostly accepting defaults — type `yes` to the terms and press
  enter through the prompts until the server initializes. (paraphrase)
- After setup you can no longer log in as root; log back in as the `openvpnas` user.
  (paraphrase)
- Run `sudo passwd openvpn` to set the password for the `openvpn` user — this is both
  the admin password and the client login for the VPN portal. (paraphrase)
- Access the admin portal at `https://<public-ipv4>:943/admin`; the self-signed
  certificate warning is expected — click advanced/proceed. Log in as `openvpn` with
  the password you set. (paraphrase)
- In Configuration → VPN Settings → Routing, set "should client internet traffic be
  routed through the VPN" to **yes**, save settings, then **update running server** so
  the change takes effect — this ensures all your traffic goes through the VPN
  encrypted. (paraphrase)
- Clients connect via the **user portal** at `:943` (drop the `/admin` path), using the
  same OpenVPN username and password; download the client for your device (Windows,
  iOS, Android), which comes pre-configured with your profile, then connect. (paraphrase)
- Verify it works with a "what's my IP" search — it should show the server's IP/location
  (France in his demo), not your home IP, confirming traffic is tunneled. (paraphrase)
- Conceptually the build does two things: gives you secure access to your AWS **Virtual
  Private Cloud (VPC)** via devices' private IPs (not public IPs), and encrypts all your
  internet traffic so you're safe wherever you are — and can appear wherever the server
  is. (paraphrase)
- It is legitimately free: an AWS free-tier account includes 12 months of free-tier
  access; one server can run 750 hours/month free, and a 31-day month is only 744 hours,
  so an always-on server stays within the free allowance. (paraphrase)
- After 12 months it's no longer free — around **$10/month** to run the server — which
  Chuck says is still competitive with other VPN options. (paraphrase)
- You can do the same on other cloud providers, but Chuck is currently partial to AWS.
  (paraphrase)
- Learning AWS / the cloud is a valuable skill; he offers an AWS Solutions Architect
  Associate course made with Anthony Sacco and David Bombal as a path to becoming a
  cloud/AWS engineer. (paraphrase)
