# never leave the terminal

## Summary

A "Five Minute Friday" episode in which Chuck sets up and tries **`links`**, a fully
text-based web browser that runs entirely inside the terminal. He installs it live on
Ubuntu under WSL and browses without a mouse, driven purely by keyboard. The appeal is
his terminal-first instinct: "I don't have to leave my terminal — I'm already there."
He walks through the navigation model (arrow keys to move/follow links, `g` to enter a
URL, `o` for options), notes that `links` blocks ads by default because it literally
can't render them, and demonstrates its limits: anything JavaScript-heavy — YouTube,
login forms, Gmail, his own course site — renders poorly or fails. He praises it as a
fun, lightweight, distraction-free way to read text-heavy sites (news, articles) from
inside the shell, while conceding it's a novelty he'd "probably end up using every once
in a while."

## Key claims (dated — tools + philosophy)

All dated 2024-09-06 (paraphrase unless quoted):

- The single tool featured is **`links`**, a text-based / terminal web browser, described
  as "a fully featured worldwide web browser" originally made by the University of Kansas.
- Installation is via the distro package manager: `apt` on Debian/Ubuntu, `dnf` on Fedora,
  `pacman` on Arch; Chuck installs it on Ubuntu under WSL with `sudo apt install links -y`.
- Navigation is keyboard-only: up/down arrows to move, right arrow to follow a link, left
  arrow to go back; `g` opens a prompt to go to a URL; `o` opens options (including a vi mode).
- You can also launch straight to a site from the shell, e.g. `links youtube.com`.
- `links` blocks ads by default because it's text-only and legitimately can't load them.
- Cookie prompts appear per-site; Chuck sets "accept all cookies" in the options menu to
  stop the repeated prompts, then saves via "accept changes."
- Limitation: because it's a text-only browser, JavaScript-heavy sites and login/forms —
  YouTube, Gmail, his ACI Learning login, his own graphics-heavy academy site — don't work
  well or at all. He recommends a normal browser for anything requiring login.
- There is a Windows version, but it targets an old Windows 7-era build; Chuck declines it
  because WSL already gives him Linux, and sticks with Linux.
- Terminal-first philosophy: the whole draw is not having to leave the terminal — he's
  "already there," so browsing from the shell keeps him in one environment.
