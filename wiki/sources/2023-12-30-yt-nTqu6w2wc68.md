# you need to learn tmux RIGHT NOW!!

## Summary

A signature "you need to learn X RIGHT NOW" tutorial teaching **tmux** (terminal
multiplexer). Chuck opens with the hook use case — SSH into a cloud Linux server with
one command, launch panes/windows, detach from a running SSH session, drive home, and
reattach to the exact same session from a different computer — calling it "a superpower."
He defines tmux as a program that runs inside your terminal and lets other terminal
programs run inside it ("terminal running inside your terminal"), then teaches by doing:
spinning up an Ubuntu VM on Akamai/Linode (the sponsor), starting a session, and building
up through tmux's three layers — **sessions, windows, panes**. Along the way he covers
detach/reattach, naming and killing sessions, splitting panes ("hot dog style" horizontal
vs "hamburger style" vertical), the `Ctrl+B` prefix key concept, moving/resizing panes,
preset layouts, windows, the `Ctrl+B w` session/window tree, killing panes/windows, a
minimal `~/.tmux.conf` (mouse on + vi mode-keys), and copy mode. Throughout he leans on
his running "cups of coffee" ping gag and Harry Potter / persona bits (Bob, Stacy). The
closing message: even if all you use is attach/detach, that alone is a win.

## Key claims (dated — 2023-12-30)

- **tmux is a terminal multiplexer** — a program that runs inside your terminal and
  allows other terminal programs to run inside of it (paraphrase).
- **The killer use case is detach/reattach**: you can run something in your terminal,
  detach from it, and reattach later from somewhere else — it keeps running even if you
  lose your internet connection or go home for the day (paraphrase).
- **tmux is often preinstalled**; where it isn't, install it with `apt install tmux`
  (paraphrase).
- **Start a session** by typing `tmux` and hitting Enter (paraphrase).
- **Detach** from a session with the keybind `Ctrl+B` then `d` — this detaches without
  ending/killing anything running inside (paraphrase).
- **Reattach** to the most recent session with `tmux a` (`a` = attach) (paraphrase).
- **Create a named session** with `tmux new -s <name>` (e.g. `tmux new -s Bob`); the name
  shows in the bottom-left status bar (paraphrase).
- **List running sessions** with `tmux ls` (paraphrase).
- **Attach to a specific session by index/target** with `tmux a -t <target>` (e.g.
  `tmux a -t 0` for session index zero) (paraphrase).
- **Kill a specific session** with `tmux kill-session -t <name>`; `tmux kill-session`
  alone kills the most recent session (paraphrase).
- **tmux has three layers**: sessions (top), windows (middle), and panes (bottom);
  creating a session automatically creates a window (paraphrase).
- **A window defaults to the name of the program running in it** (e.g. `bash`, index
  zero) and fills the whole terminal area (paraphrase).
- **Split a pane horizontally ("hot dog style")** with `Ctrl+B %` (paraphrase).
- **Split a pane vertically ("hamburger style")** with `Ctrl+B "` (double quote)
  (paraphrase).
- **`Ctrl+B` is the prefix key** — pressing it tells tmux a command is coming; any keybind
  after it is the action. Chuck keeps it at the default so tmux behaves the same on every
  system he installs it on (paraphrase).
- **Documentation notation `C-b`** means `Ctrl+B` (or whatever your prefix key is)
  (paraphrase).
- **Move between panes** with `Ctrl+B` then a directional arrow key (up/down/left/right);
  if fast enough you can keep moving before the prefix times out (paraphrase).
- **Show pane indexes and jump to one** with `Ctrl+B q`, then the pane's number
  (e.g. `Ctrl+B q 2`) — you have to be quick (paraphrase).
- **Resize the active pane** with `Ctrl+B` while holding `Ctrl` + directional arrows;
  hold `Alt` instead of `Ctrl` for bigger resize steps (paraphrase).
- **Apply preset layouts** with `Ctrl+B` then `Alt+1` through `Alt+5` (five preset
  layouts) (paraphrase).
- **Create a new window** with `Ctrl+B c`; the current window is marked with an asterisk
  in the status bar (paraphrase).
- **Move sequentially through windows** with `Ctrl+B n` (paraphrase).
- **Rename the active window** with `Ctrl+B ,` (comma) (paraphrase).
- **Open the session/window tree** with `Ctrl+B w` — lists all windows AND all sessions
  so you can preview and jump into any of them with Enter (Chuck's preferred way to move
  around) (paraphrase).
- **Kill the active pane** with `Ctrl+B x` (confirm with `yes`) (paraphrase).
- **Kill the whole window** with `Ctrl+B &` (ampersand) (confirm with `yes`) (paraphrase).
- **Kill the entire tmux server** (all sessions at once) with `tmux kill-server`
  (paraphrase).
- **Recommended `~/.tmux.conf` tweaks**: add `set -g mouse on` (mouse support) and
  `set -w -g mode-keys vi` (enables vi keys in copy mode) (paraphrase).
- **Copy mode from the keyboard**: enter with `Ctrl+B [` (opening bracket), move to the
  start point and press Space to begin selecting, move with arrows (or vi keys h/j/k/l)
  to the end, press Enter to copy into the buffer, then paste with `Ctrl+B ]` (closing
  bracket) (paraphrase).
- **With mouse mode on**, you can also just highlight text with the mouse to enter copy
  mode and copy — but Chuck prefers keyboard-only so your hands never leave the keyboard
  (paraphrase).
- **Practical framing**: tmux makes you hyper-productive in Linux for programming or
  hacking — e.g. run an Nmap scan in one pane, Metasploit in another, and a subdomain
  enumeration tool (sub-finder) detached in another (paraphrase).
