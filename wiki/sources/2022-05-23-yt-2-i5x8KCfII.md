# i bet you can't do this (because you still suck at subnetting)

## Summary

An entry in Chuck Keith's "You SUCK at Subnetting" series, this video teaches
binary–decimal conversion as a foundational, prerequisite skill for subnetting.
Chuck frames binary as "the matrix code" behind IP addresses: the same address a
human reads in dotted-decimal is what the computer sees as 32 ones and zeros.
He builds up the mental model (bits, bytes, octets), introduces a "powers of two"
chart as the core tool ("secret decoder ring"), then walks through converting
binary → decimal for a full IP address, and decimal → binary in reverse using
subtraction. The pedagogical throughline: this is a "leg day" building block you
must practice and cement before you can master subnetting. Includes two ad reads
(SafeDNS, Manscaped).

## Key claims (dated 2022-05-23 unless noted; all paraphrase)

- An IP address is 32 bits total: four octets of 8 bits each (8 × 4 = 32). In bytes,
  an IP address is 4 bytes.
- Binary is "the language of computers" — only ones and zeros. A `1` means the bit is
  "on," a `0` means "off," like a light switch.
- Storage sizes descend gigabyte → megabyte → kilobyte → byte → bit; a bit is the
  lowest/smallest unit, "the ground floor."
- 8 bits make 1 byte. The word "octet" comes from "octo" (eight) — each section
  between the dots holds eight bits.
- The core tool is a "powers of two" chart: write 1, 2, 4, 8, 16, 32, 64, 128
  (written right-to-left / smallest on the right, so the leftmost bit is 128). Each
  value is 2 raised to a power (2^0 = 1, 2^1 = 2, 2^2 = 4, … 2^7 = 128).
- You can build the chart two ways: (1) take powers of two from 2^0 up to 2^7 (eight
  digits total), or (2) start at 1 and keep multiplying by 2.
- **Binary → decimal method:** line up the 8 bits of an octet under the powers-of-two
  chart. For every bit that is "on" (1), write down its column value; ignore bits that
  are "off" (0). Add the on-values together to get the decimal value of the octet.
  - Worked examples from the video's IP address: `11000000` = 128 + 64 = 192;
    `10101000` = 128 + 32 + 8 = 168; `00000001` = 1; `00010101` = 16 + 4 + 1 = 21
    (final address ends `…192.168.1.21`-style octets: 192, 168, 1, 21).
- **Decimal → binary method (subtraction):** take the octet's decimal value and move
  left-to-right across the chart (128, 64, 32, …). For each column ask "can I subtract
  this value from the remaining number?" If yes, that bit is "on" (1) and you subtract
  it; if no, the bit is "off" (0). Continue until the remainder reaches 0; all
  remaining bits are 0.
  - Worked examples: 172 → `10101100` (172−128=44, 44−32=12, 12−8=4, 4−4=0);
    16 → `00010000`; 34 → `00100010` (34−32=2, 2−2=0); 3 → `00000011` (3−2=1, 1−1=0).
- These conversions are essential to subnetting — "leg day," a building block not to be
  skipped. Mastery requires repeated practice; extra practice problems are offered at
  NetworkChuck Academy.
